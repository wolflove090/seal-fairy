# 妖精コレクション未発見画像transparent表示要件書

## 背景と目的
- 現在の妖精コレクションでは、未発見の妖精カードに対して `HudScreenBinder.CreateFairyCard()` 内で `fairy-card__image--undiscovered` クラスを付与し、USS 側の見た目調整のみで未発見状態を表現している。
- 今回は未発見妖精の画像欄に、空見た目ではなく `transparent` 素材を明示的に表示し、未発見カードの見た目を統一する。
- 既存の発見状態管理、カード件数、名前の伏せ字、開閉導線は維持したまま、未発見画像の表示仕様だけを変更する。

## スコープ内
- 妖精コレクションカードの未発見時画像表示仕様の変更
- `transparent` 素材を用いた未発見カード画像欄の表示ルール定義
- 既存のカード生成処理と UI 見た目定義の責務分担整理
- 未発見画像差し替え後の確認観点整理

## スコープ外
- 発見済み妖精カードの画像表示仕様変更
- 妖精名、補足文、発見数表示文言の変更
- 妖精獲得状態の保存方式変更
- 新規妖精データ、詳細画面、フィルタ、ソートの追加
- 妖精コレクション画面全体のレイアウト刷新

## 機能要件

### 1. 基本表示
- 妖精コレクション画面は `FairyCatalogSource.GetFairies()` に含まれる全妖精カードを従来どおり表示する。
- 発見済み妖精は `FairyDefinition.Icon` をそのまま表示する。
- 未発見妖精は画像欄に `transparent` 素材を表示する。
- `transparent` 素材は既存プロジェクト内の `Assets/GameResources/Texture/Resources/transparent.png` を利用対象とする。

### 2. 未発見カード表示
- 未発見妖精のカードは、画像欄の背景が空になるのではなく、`transparent` 素材が設定された状態で表示される。
- 未発見妖精名は従来どおり `？？？` を表示する。
- 未発見時の補足文表示が既存で `未発見` またはそれに準じた伏せ表示を採用している場合、その仕様は維持する。
- 発見済みと未発見でカードサイズ、画像フレームサイズ、グリッド配置は変えない。

### 3. フォールバック
- `transparent` 素材の取得に失敗した場合でも、例外で画面更新が停止しない。
- 素材取得失敗時は、少なくとも現在と同等の未発見表示を維持できる構成にする。
- 発見済み画像が未発見カードに誤って表示されないことを保証する。

### 4. 既存実装との整合
- 妖精コレクションの開閉は既存の `OpenFairyCollection()` と `CloseFairyCollection()` を維持する。
- 一覧更新は既存の `RefreshFairyCollection()` を維持する。
- 未発見判定は既存どおり `FairyCollectionService.IsDiscovered()` を用いる。
- 妖精マスターデータ `FairyDefinition` に新しい serialized field は追加しない。

## 非機能要件
- UI Toolkit ベースの構成を維持する。
- 既存の妖精コレクション画面表示性能を大きく悪化させない。
- 画像差し替え処理は、妖精件数増加時も理解しやすい責務分離を保つ。
- 素材参照方法は、後続で別の未発見画像へ差し替える際に追跡しやすい形にする。

## 受け入れ条件
- 妖精コレクション画面を開いたとき、発見済み妖精カードは従来どおり個別アイコンを表示する。
- 未発見妖精カードの画像欄に `transparent` 素材が表示される。
- 未発見妖精カードで `？？？` 表示が維持される。
- 発見済み件数表示 `X/Y` とカード並び順は変更されない。
- 妖精コレクション画面の開閉、スクロール、ショップとの排他表示に回帰がない。
- `transparent` 素材が未設定または取得失敗しても、コレクション画面更新時にエラー停止しない。

## 未確定事項・確認事項
- 未発見カードの補足文表示は、現行実装の固定文言 `クール/ワイルド` を維持するか、過去資料どおり `未発見` に寄せるかを実装計画時に確認する。
→ 現行実装を維持します
