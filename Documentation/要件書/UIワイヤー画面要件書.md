# UIワイヤー画面要件書

## 背景と目的
- 添付ワイヤーをベースに、ゲーム画面へ常時表示する HUD を UI Toolkit で構築する。
- 今回の対象は UI の見た目とレイアウトのみとし、ボタン押下時の挙動、データ反映、状態遷移は実装しない。
- 実装作業はユーザー側で行うため、本書ではユーザーがそのまま作業できる粒度で構成と作業指示を定義する。

## スコープ内
- UI Toolkit 用の UXML / USS / Panel Settings / UIDocument の構成決定
- ワイヤーにある 4 要素の画面固定レイアウト定義
- プレースホルダー文言を使った静的 UI の作成指示
- 再利用しやすい `name` / class 命名方針の定義
- 16:9 の 1920x1080 を基準にしたレイアウト定義

## スコープ外
- ボタンのクリック処理
- 金額や状態文言の動的更新
- 妖精ボタン、ショップボタン、準備完了ボタンの画面遷移
- アニメーション、効果音、入力制御
- C# スクリプトによる UI バインド

## 機能要件

### 1. 実装方式
- UI は Unity の UI Toolkit で実装する。
- シーン `Assets/Main.unity` で利用できる構成にする。
- 画面表示は `UIDocument` を使い、ゲームワールドとは独立したスクリーン固定 UI とする。
- UI の見た目は UXML と USS で完結させ、今回の段階では C# スクリプトを追加しない。

### 2. 画面構成
- 画面内に以下 4 ブロックを配置する。
- 左上: 所持金表示パネル
- 右上: 準備完了ボタン
- 右下上段: 妖精ボタン
- 右下下段: ショップボタン
- ワイヤー上の「準備完了」は右上固定のボタンとして扱う。

### 3. レイアウト要件
- 16:9 の 1920x1080 を基準解像度として作成する。
- ルート要素は画面全体にストレッチさせる。
- 所持金表示パネルは左上に固定し、画面端から十分な余白を取る。
- 準備完了ボタンは右上に固定し、所持金表示パネルと同じ上マージンに揃える。
- 右下の 2 ボタンは縦積みし、画面右端と下端から十分な余白を取る。
- 妖精ボタンとショップボタンの幅は揃える。
- 妖精ボタンとショップボタンの間には明確な縦間隔を設ける。
- 解像度が変わっても、4 要素の相対位置が崩れないようにアンカー相当のレイアウトにする。

### 4. 表示文言
- 所持金表示は `お金：XXX円` を初期表示文言にする。
- 右上ボタンは `準備完了` を初期表示文言にする。
- 右下上段ボタンは `妖精` を表示する。
- 右下下段ボタンは `ショップ` を表示する。
- すべての文言は日本語で表示する。

### 5. 見た目
- 背景色はワイヤーどおり白基調とする。
- 各パネルとボタンは薄いグレーの塗りで統一する。
- 角丸は小さめに付け、ワイヤーの柔らかい矩形に寄せる。
- 枠線は必須ではない。必要なら薄いグレー系 1px 程度に留める。
- 文字色は黒とし、視認性を優先する。
- フォントサイズはワイヤーに近い大きめ設定にし、小画面でも読めることを優先する。
- 所持金表示パネルは他要素より横長にする。
- 右上ボタンと右下の 2 ボタンは、タップしやすい大きさを維持する。

### 6. 推奨 UI 構造
- ルート直下に全画面用のコンテナを置く。
- 上段用コンテナと右下用コンテナを分ける。
- 上段用コンテナは左右端揃えの横並びにする。
- 右下用コンテナは画面右下固定の縦並びにする。
- 所持金表示は `Label` を内包した `VisualElement` で構成する。
- 準備完了、妖精、ショップは `Button` を使う。

### 7. 命名要件
- UXML 上の `name` は以下を使う。
- `root`
- `top-bar`
- `money-panel`
- `money-label`
- `ready-button`
- `bottom-right-menu`
- `fairy-button`
- `shop-button`
- USS class は役割ベースで付与し、見た目調整を class 側へ寄せる。

## 非機能要件
- 既存のゲームロジック実装へ干渉しない。
- UI 実装は後から文言差し替えやイベント接続がしやすい構成にする。
- 1 ファイルに見た目を詰め込みすぎず、少なくとも UXML と USS は分離する。
- Unity 6000.3.3f1 環境で編集しやすい標準的な UI Toolkit 構成にする。

## ユーザー作業指示
1. `Assets/UI/` 配下に UI 用フォルダを新規作成する。
2. `Assets/UI/UXML/` に画面用 UXML を作成する。
3. `Assets/UI/USS/` に画面用 USS を作成する。
4. `Assets/UI/Settings/` に Panel Settings アセットを作成する。
5. `Assets/Main.unity` 上に HUD 表示用 GameObject を追加し、`UIDocument` をアタッチする。
6. `UIDocument` へ作成した UXML を割り当て、Panel Settings も接続する。
7. UXML には本書の命名要件に沿って要素を配置する。
8. USS で白背景、薄グレーのパネルとボタン、右下 2 連ボタン、上段左右配置を調整する。
9. 文言はすべて固定値で入れ、イベントやデータバインドは設定しない。
10. Game ビューで 1920x1080 の 16:9 を基準に確認し、左右上端と右下端の余白が崩れないことを確認する。
11. 縦横比が変わっても右下メニューが画面外に出ないことを確認する。

## 受け入れ条件
- `Assets/Main.unity` を開くと、HUD が画面固定で表示される。
- 1920x1080 の Game ビューで、左上に `お金：XXX円`、右上に `準備完了` ボタンが見える。
- 右下に `妖精` と `ショップ` の 2 ボタンが縦並びで見える。
- UI はカメラやワールドオブジェクトの移動の影響を受けない。
- 4 要素ともワイヤーに近い位置関係で表示される。
- 現時点でクリック処理や値更新処理は接続されていない。

## 未確定事項・確認事項
- 最終デザインで使用する正式フォントは未確定のため、当面は Unity 標準フォントでよい。
- パネルとボタンの厳密なピクセル寸法は未確定のため、ワイヤー準拠の比率を優先して調整する。
