# 所持金付きシールショップ機能要件書

## 背景と目的
- 現在のシールショップはシール購入時に所持シールを増やせるが、通貨の概念がなく、HUD 左上の所持金表示もプレースホルダーのままである。
- ショップでの購入判断にゲーム内リソース管理を持たせるため、所持金の保持、HUD 反映、購入時の減算、購入不可状態の視覚化を追加する。
- シールごとに価格を設定できるようにし、今後のシール追加時も Inspector 上で価格調整できる構成を定義する。
- 初回の所持金を 1000 円とし、プレイヤーが現在の所持金で購入可能なシールだけを選べるショップ体験を成立させる。

## スコープ内
- ゲーム開始時の初期所持金を 1000 円にする仕様
- HUD 左上の所持金表示を実データに連動させる仕様
- シールごとに価格を設定し、ショップ一覧に反映する仕様
- シール購入時に所持金を価格分だけ減算する仕様
- 所持金不足のシールを購入不可とし、ショップ内でグレーアウト表示する仕様
- 購入成功後に所持シール一覧と所持金表示を即時更新する仕様

## スコープ外
- 所持金の永続化
- 購入確認ダイアログ
- 購入時の演出、SE、アニメーション
- 所持金の獲得手段追加
- 割引、セール、複数通貨
- 課金、バックエンド連携
- 売却、返金、購入履歴表示

## 機能要件

### 1. 所持金データ管理
- ゲーム開始時の所持金は 1000 円とする。
- 所持金は実行中に参照・更新できる単一のランタイムデータとして管理する。
- 所持金は 0 円未満になってはならない。
- 所持金更新時は、所持金表示 UI と購入可能状態の再評価を同一フローで行う。

### 2. HUD 左上の所持金表示
- HUD 左上の `money-label` は実際の所持金を `お金：{金額}円` 形式で表示する。
- ゲーム起動直後に `お金：1000円` と表示されること。
- シール購入成功後、減算後の金額へ即時更新されること。
- 所持金が変化しない操作では表示も変化しないこと。

### 3. シール価格設定
- 各シールは個別の価格を持つ。
- 価格はシール定義ごとに Inspector で設定できること。
- 価格は 0 以上の整数で扱う。
- ショップ一覧は各シールカードに価格を表示する。
- 価格未設定による曖昧状態を避けるため、全販売シールに価格が設定されている前提とする。

### 4. ショップ一覧表示
- ショップ一覧は `StickerShopCatalogSource` に設定された順で表示する。
- 各カードには少なくともシール画像、シール名、価格を表示する。
- 現在の所持金で購入可能なシールは通常表示とする。
- 現在の所持金で購入不可なシールはグレーアウト表示とし、購入できないことが視覚的に分かること。
- 一覧 0 件時は空表示メッセージを表示し、例外を発生させないこと。

### 5. 購入可否判定
- 購入可否は `所持金 >= シール価格` で判定する。
- 所持金不足のカードはクリックできない、またはクリックしても購入処理を実行しないこと。
- 所持金不足カードはグレーアウト状態と操作不可状態を常に同期させること。
- 所持金変動後は全カードの購入可否を再評価し、見た目と操作状態を更新すること。

### 6. 購入処理
- プレイヤーが購入可能なシールカードを選択した時点で購入を成立とする。
- 購入成功時は、対象シールの価格分だけ所持金を減算する。
- 購入成功時は、対象シールを所持シールへ 1 件追加する。
- 購入成功後もショップ画面は閉じず、続けて購入できること。
- 購入成功時には少なくとも Console に対象シール名、価格、購入後残高が分かるログを出力すること。
- 購入失敗時は所持金減算も所持シール追加も行わないこと。

### 7. 所持シール反映
- 購入したシールは既存の所持シール一覧に即時反映されること。
- 所持シール一覧が空から復帰した場合、空表示ラベルを非表示に切り替えること。
- 既存仕様どおり、新規購入シールは所持一覧の先頭に追加すること。
- 既に別シールを選択中の場合、購入後も現在選択中のシールは維持すること。

### 8. 重複購入
- 同じシールは複数回購入可能とする。
- 重複購入時は、購入回数に応じて所持数を増やす。
- 重複購入時も都度その価格分の所持金を減算する。
- 価格はシール定義単位で固定とし、同じシールの購入回数によって変動しない。

### 9. UI 整合性
- ショップ画面と妖精コレクション画面の排他表示は既存仕様を維持する。
- ショップ画面を開いた時点で、ショップ内の所持金表示も最新残高を表示する。
- HUD 左上表示とショップ内表示が共存する場合、同じ残高を示すこと。
- 購入直後に HUD とショップ内表示のどちらかだけが古い値のまま残らないこと。

## 非機能要件
- 既存のシール配置フェーズ、シールめくりフェーズ、妖精コレクション画面の挙動を壊さない。
- 所持金不足カードが含まれる状態でも null 参照例外や UI 再構築例外を発生させない。
- 購入処理は 1 クリックにつき 1 回だけ成立し、多重減算を起こしにくい構成にする。
- 価格設定フィールドやメソッド名は、所持シールデータと価格データの責務差が分かる命名にする。
- UI Toolkit ベースの既存画面構成を維持し、大きな画面構造変更を避ける。

## 受け入れ条件
- ゲーム起動直後、HUD 左上に `お金：1000円` が表示される。
- 各販売シールに価格が表示される。
- 所持金以上の価格を持つシールはグレーアウト表示され、購入できない。
- 所持金以下の価格を持つシールは通常表示され、購入できる。
- 1000 円の所持金で 300 円のシールを購入した場合、購入後に `お金：700円` と表示される。
- 購入成功時、対象シールが所持一覧に 1 件追加される。
- 購入成功後、残高不足になった別シールがあれば、その場でグレーアウトへ更新される。
- 残高不足のシールを押しても、所持金と所持シール一覧は変化しない。
- 同じシールを複数回購入すると、購入回数分だけ所持数が増え、その都度価格分の所持金が減る。
- ショップ画面は購入後も開いたままで、`閉じる` または背景押下で閉じられる。
- HUD 左上表示とショップ内所持金表示が同じ値を示す。

## 未確定事項・確認事項
- 所持金の獲得手段は今回スコープ外のため、テスト中の残高回復は Inspector 初期値変更または再起動で対応する想定。
- 購入不可カード押下時のフィードバックは、今回グレーアウトのみで十分とする。
- 価格を `StickerDefinition` に直接持たせるか、ショップ専用データに分離するかは実装設計で決定する。
→ StickerDefinitionに持たせます
- 所持金表示は HUD 左上とショップフッターの 2 箇所に出る前提で整理しているが、最終 UI は既存レイアウトとの兼ね合いで調整余地がある。
