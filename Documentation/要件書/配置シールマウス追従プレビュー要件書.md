# 配置シールマウス追従プレビュー要件書

## 背景と目的
- 現在の `TapStickerPlacer` は選択中シールをクリックまたはタップ位置へ直接配置するが、配置前に「どのシールがどこへ置かれるか」を画面上で確認できない。
- `OwnedStickerInventorySource` ではシール残数を管理しており、一覧 UI では各シールの所持数を表示しているが、配置操作中に選択中シールの残数を即座に把握しづらい。
- シール配置フェーズ中、選択中シールをマウス位置へ追従表示し、あわせて残数も表示することで、配置対象と在庫状況を同時に把握できる操作体験にする。

## スコープ内
- `SealPlacement` フェーズ中のみ有効な、選択中シールのマウス追従プレビュー表示を追加する。
- プレビューと同時に、選択中シールの現在残数を画面上へ表示する。
- 選択シール変更、残数変動、フェーズ切り替えに応じてプレビュー内容と残数表示を同期する。
- プレビュー表示が既存のシール選択 UI、配置処理、UI 操作抑止ルールと矛盾しないようにする。

## スコープ外
- シール一覧パネル自体のレイアウト全面改修
- シール残数の保存、購入、復元など在庫管理仕様の変更
- プレビューへのアニメーション、回転、透明度補間などの演出追加
- タッチ専用 UI 最適化や長押し・ジェスチャー対応
- 配置可能 / 不可能領域の新規判定ロジック追加

## 機能要件

### 1. 対象フェーズ
- 本機能は `SealPlacement` フェーズでのみ動作する。
- `SealPeeling` フェーズ中はプレビューシールと残数表示を非表示にする。
- `SealPeeling` から `SealPlacement` へ戻った直後、選択中シールが未設定であればプレビューと残数表示は出さない。

### 2. プレビュー表示
- 選択中シールが存在する場合、選択中シールと同一見た目のプレビューをマウス位置へ追従表示する。
- プレビューは「これから配置されるシール」を示す見た目とし、実配置前の仮表示として扱う。
- プレビュー位置は、既存の配置処理が使用しているポインタ位置と同じ入力系を基準に更新する。
- プレビューに使用する見た目は、実際に配置される `StickerDefinition.StickerPrefab` と一致させる。
- 選択中シールが切り替わった場合、プレビューの見た目も即座に切り替える。
- 選択中シールの残数が 0 になった場合、プレビューは非表示にする。

### 3. 残数表示
- プレビュー付近に、選択中シールの残数を数値で表示する。
- 残数表示は `OwnedStickerInventorySource` の現在値を参照する。
- シール配置が成功して残数が減少した場合、表示値も即時更新する。
- シールを切り替えた場合、残数表示は新しい選択中シールの値へ即時更新する。
- 選択中シールが存在しない場合、残数表示は非表示にする。
- 残数表示の内容は、プレイヤーが残り枚数と分かる文言または表記にする。

### 4. 配置操作との連携
- プレビューが表示されている位置と、実際にクリックまたはタップで配置される位置は一致すること。
- UI 上を操作中は、既存ルールどおりワールドへの配置を行わない。
- UI 上を操作中に、プレビューの表示位置が不自然に更新され続ける場合は、UI 操作と干渉しない振る舞いに統一する。
- 配置成功後、残数が残っている場合は同じシールのプレビュー表示を継続する。
- 配置成功後、残数が 0 になった場合はプレビューと残数表示を非表示にする。

### 5. 既存選択 UI との整合
- 画面左下のシール一覧セルに表示している所持数表示は継続する。
- 今回追加する残数表示は、一覧セル上の所持数表示を置き換えるものではなく、配置中の補助表示として扱う。
- シール一覧で未選択状態のときは、一覧セルの選択表示、プレビュー表示、残数表示がすべて未選択状態として整合する。
- 一覧内のシールを再選択した際は、既存の選択強調表示と新しいプレビュー表示の両方が同じ対象を示す。

### 6. UI/表示ルール
- プレビューと残数表示は、カメラ移動の影響を受けず、プレイヤーのポインタ操作に追従する。
- 残数表示はプレビューやカーソルと重なりすぎず、視認できる位置に出す。
- 既存の HUD 要素である左上所持金、右上準備完了、左下シール一覧、右下ショップ / 妖精ボタンの視認性を大きく損なわない。
- 1920x1080 を基準に、シールの見た目と残数表示を無理なく確認できるサイズとする。
- 解像度やアスペクト比が変わっても、プレビューと残数表示の相対配置が大きく崩れないこと。

## 非機能要件
- 既存の `TapStickerPlacer`、`HubScreenBinder`、`StickerSelectionState`、`OwnedStickerInventorySource` の責務を極端に肥大化させない構成にする。
- プレビュー表示のために、毎フレーム不要な Instantiate / Destroy を繰り返さない。
- 残数表示の更新は、既存の在庫変更イベントまたは同等の同期ポイントを利用して、状態不整合を起こしにくい構成にする。
- マウス操作時の見た目更新が著しく重くならないこと。
- Inspector で参照差し替えや表示パラメータ調整ができる構成を優先する。

## 受け入れ条件
- `SealPlacement` フェーズ中に選択中シールがある場合、マウス位置に選択中シールのプレビューが表示される。
- プレビュー付近に、選択中シールの残数が同時に表示される。
- シール一覧から別シールを選ぶと、プレビュー見た目と残数表示が即時に切り替わる。
- シールを 1 枚配置すると、同シールの残数表示が 1 減る。
- 残数が 0 になった時点で、そのシールのプレビューと残数表示が消える。
- シール未選択状態では、プレビューと残数表示がどちらも出ない。
- `SealPeeling` フェーズへ遷移すると、プレビューと残数表示が非表示になる。
- UI 操作中に意図せずシール配置が発生しない。

## 未確定事項・確認事項
- 残数表示は、画面固定 HUD ではなくカーソル近傍のワールド追従表示として実装する。
- 本機能ではモバイル向け表示最適化は考慮対象外とし、PC のマウス操作を前提に設計する。
