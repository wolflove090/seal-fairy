# シールショップ機能要件書

## 背景と目的
- 既存 HUD には `ショップ` ボタンが存在するが、押下後に表示するシールショップ画面は未実装である。
- 現在の実装には HUD、シール一覧、妖精コレクションの UI パターンが存在するため、それらを踏襲して「シールを一覧で見られるショップ画面」を追加したい。
- 今回は購入処理までは行わず、ショップ画面を開き、シール一覧をスクロール表示し、各シールタップ時にシール名をログ出力できる状態を定義する。

## スコープ内
- `OwnedStickerDefinition` を保持シールとショップ表示の共通定義として扱える形へリファクタリングすること
- HUD の `ショップ` ボタン押下で開くシールショップ画面の仕様定義
- 添付ワイヤーをベースにしたショップ画面レイアウトの定義
- 妖精コレクション UI と同系統のオーバーレイ挙動、閉じる操作、スクロール一覧仕様の定義
- シール一覧アイテムの最低限の表示項目定義
- シールタップ時にシール名を `Debug.Log` 出力する挙動の定義

## スコープ外
- シール購入処理
- 所持金の増減処理
- 購入可能 / 購入済み / 売り切れなどの状態管理
- シール詳細画面への遷移
- ショップ内の並び替え、カテゴリ分け、検索、ページング
- バックエンド連携、永続化、課金機能

## 機能要件

### 1. 画面起動と終了
- HUD 右下の `ショップ` ボタン押下でシールショップ画面を表示する。
- ショップ画面は既存 HUD の上に重なる全画面オーバーレイとして表示する。
- ショップ画面には `閉じる` ボタンを配置し、押下で画面を閉じる。
- 妖精コレクション UI と同様に、背景オーバーレイ押下でもショップ画面を閉じる。
- ショップ画面を閉じた後は、元の HUD 表示状態へ戻る。

### 2. 画面構成
- 画面左側には既存 HUD が見える状態を維持しつつ、ショップ画面本体はワイヤーどおり右側の大型パネルとして表示する。
- ショップパネル上部にタイトル `シールショップ` を表示する。
- パネル内中央から下部にかけて、シール一覧を縦スクロールで表示できる領域を配置する。
- パネル下部には `閉じる` ボタンと所持金表示を配置する。
- 背景オーバーレイは背面ゲーム操作と背面 HUD 操作を遮断する。

### 3. 一覧表示対象
- 一覧にはショップで提示する全シールを表示する。
- 初期表示順はデータソースに登録された順とする。
- 一覧対象が 0 件のときは、空表示メッセージを表示し、例外で停止しない。
- 一覧データは UI 側で直書きせず、ショップ専用のカタログから取得する構成にする。
- ショップ専用カタログが保持するシール情報は、保持シール一覧と共通の `OwnedStickerDefinition` を用いる。

### 4. 一覧レイアウト
- 一覧は縦スクロール可能にする。
- 一覧アイテムはワイヤーを基準に 3 列グリッドを基本とする。
- 画面幅やパネル幅の都合で崩れないよう、折り返し可能なレイアウトにする。
- 各アイテムカードには少なくとも以下を含める。
- シール画像
- シール名
- 将来購入情報を載せられる余白または拡張しやすい構造
- 一覧項目数が増えても、パネル高さを超える分はスクロールで最後まで閲覧できること。

### 5. シールカードの表示内容
- シール画像はショップ専用カタログが参照する `OwnedStickerDefinition` に紐づく `Sprite` または同等の画像アセットを使用する。
- シール名は共通シール情報 `OwnedStickerDefinition` が保持する表示名を表示する。
- 現時点では価格表示、購入ボタン、所持状態表示は必須としない。
- シール画像未設定でも例外で停止せず、プレースホルダー表示または空枠表示で継続できること。

### 6. シールタップ時の挙動
- 一覧の各シールカードまたはシール画像タップ時に、そのシール名を `Debug.Log` で出力する。
- ログ文言は、どのシールが押されたかが Console で判別できる内容にする。
- タップ時に購入処理、所持金更新、確認ダイアログ表示は行わない。
- タップ後もショップ画面は閉じず、そのまま表示を維持する。

### 7. UI 構成と実装方式
- 実装方式は既存 HUD および妖精コレクションと同じく UI Toolkit を前提とする。
- ショップ画面は既存 HUD とは分割した新規 UXML / USS で実装する。
- 表示制御は `HubScreenBinder` から行える構成にする。
- `HubScreenBinder` にはショップ画面の `VisualTreeAsset` 参照、開閉制御、一覧生成、タップ時ログ出力処理を追加できる構成にする。
- ショップ用の一覧データは、既存所持シール一覧とは分離した専用カタログから取得する。
- ショップ専用カタログは新設するが、個々のシール情報クラスは `OwnedStickerDefinition` を共通利用する。
- 実装着手は最初に `OwnedStickerDefinition` のリファクタリングを行い、その後ショップカタログと UI 実装へ進む。
- 妖精コレクションとショップ画面が同時に開かないようにする。

### 8. 既存 UI 踏襲要件
- 画面の開閉方式、背景オーバーレイ、右側パネル表示、`閉じる` ボタン配置は妖精コレクション UI を踏襲する。
- UI 要素の命名規則は既存 UXML / USS と同様に、役割が追いやすい `name` と class を用いる。
- `ScrollView` の利用方法、カード生成の責務、オーバーレイ表示制御は妖精コレクション実装を基準に揃える。
- 既存のシール一覧パネル、フェーズ切り替え、妖精コレクション表示を壊さない。

### 9. 初期表示文言
- ショップパネル下部の所持金表示は、今回は固定文言で表示する。
- 初期表示文言はワイヤー準拠で `お金：999円` 相当の固定値を使用する。
- 今回は HUD 左上の所持金表示との同期更新は行わない。

## 非機能要件
- 既存のシール配置フェーズ、シールめくりフェーズの操作を壊さない。
- `OwnedStickerDefinition` のリファクタリング後も、既存の所持シール選択 UI と配置処理が破綻しないこと。
- ショップ画面表示中は背面のゲーム入力と HUD ボタン入力を受け付けない。
- シール件数が増えても極端なレイアウト崩れなく表示できること。
- UI 要素名、クラス名、バインド箇所は後続実装で追いやすい命名にする。
- ワイヤー基準解像度は 1920x1080 とし、同等の 16:9 画面で大きく崩れないこと。
- 妖精コレクション UI と同時に存在しても、表示優先順位や参照初期化で競合しないこと。

## 受け入れ条件
- `OwnedStickerDefinition` を共通定義として使う形へリファクタリングしても、既存の所持シール一覧と配置処理が正常動作する。
- ゲーム画面で `ショップ` ボタンを押すと、シールショップ画面が表示される。
- ショップ画面にシール一覧がカード形式で表示される。
- 一覧項目が表示領域を超える場合でも、スクロールして最後まで確認できる。
- 各シールをタップすると、そのシール名が Console に出力される。
- シールタップでは購入処理や所持金更新は発生しない。
- `閉じる` ボタン、または背景オーバーレイ押下でショップ画面が閉じる。
- 一覧データ 0 件でもショップ画面は正常に開き、空表示のまま動作する。
- ショップ画面表示中は背面ゲーム操作と他 HUD 操作が発生しない。
- 妖精コレクション UI の既存挙動を壊さない。

## 未確定事項・確認事項
- `OwnedStickerDefinition` にシール名表示用フィールドが必要になるため、既存の所持シール用途へ影響しない形で項目追加する実装方法は設計時に確認する。
