# シールショップ機能要件書

## 背景と目的
- 既存 HUD には `ショップ` ボタンと所持シール一覧があるが、現状のショップ画面はシール名をログ出力するだけで、所持シール増加にはつながっていない。
- シール配置フェーズで使う所持シールは `OwnedStickerInventorySource` から取得しているため、ショップで選択したシールをこの所持データへ反映できるようにする必要がある。
- 今回は通貨や価格計算を省略し、ショップでシールを選ぶと所持シールが増える最小機能を成立させる。
- ゲーム開始時の初期所持シールを 0 枚に変更し、ショップ購入によって初めて配置可能なシールが増える体験を定義する。

## スコープ内
- 初期所持シールを 0 件で開始する所持データ初期化仕様
- ショップ画面でシールを選択した際に、対応するシールを所持シールへ追加する仕様
- 購入後に所持シール一覧 UI が更新され、配置候補として選択可能になる仕様
- 同じシールを複数回購入できるかどうかを明示した所持データ更新仕様
- ショップ画面の基本表示、閉じる操作、一覧表示、購入後の継続表示仕様
- 購入結果をユーザーが把握できる最低限の UI 反映とログ出力仕様

## スコープ外
- 所持金の減算処理
- シール価格の設定、表示、検証
- 購入確認ダイアログ
- 購入演出、アニメーション、SE
- セーブデータへの永続化
- バックエンド連携、課金機能
- 購入履歴表示、売却、返金

## 機能要件

### 1. 初期所持状態
- ゲーム開始直後の所持シール一覧は 0 件とする。
- 初期所持 0 件でもゲームは例外なく起動し、所持シール一覧 UI は空表示で継続動作する。
- 初期所持 0 件の状態では、シール配置用の選択中シールは `null` とする。
- 初回購入前に `Ready` など既存フェーズ操作を行っても、所持シール 0 件起因の例外を発生させない。
- 所持シール 0 件でも `Ready` ボタンによる既存フェーズ遷移は許可する。

### 2. ショップ画面の起動と終了
- HUD の `ショップ` ボタン押下でシールショップ画面を表示する。
- ショップ画面は既存 HUD の上に重なるオーバーレイとして表示する。
- `閉じる` ボタンまたは背景オーバーレイ押下でショップ画面を閉じる。
- ショップ画面と妖精コレクション画面は同時に開かず、片方を開く際は他方を閉じる。

### 3. ショップ一覧表示
- ショップ画面には購入可能な全シールを一覧表示する。
- 一覧順は `StickerShopCatalogSource` に設定された順とする。
- 一覧 0 件時は空表示メッセージを表示し、例外で停止しない。
- 各カードには少なくともシール画像とシール名を表示する。
- 現時点では価格表示を必須としない。

### 4. 購入操作
- プレイヤーがショップ内のシールカードを選択した時点で、そのシールを購入したものとして扱う。
- 購入時に所持金チェック、所持金減算、購入可否判定は行わない。
- 購入対象のシールは、所持シールデータへ 1 件追加する。
- 今回は同じシールの重複購入を許可し、選択するたびに所持件数を 1 件増やす。
- 購入後もショップ画面は閉じず、同一画面上で続けて別シールを購入できる。
- 購入時には Console 上で対象シール名と購入成功が分かるログを出力する。

### 5. 所持シールデータ更新
- 所持シール一覧の参照元は、実行中に件数を増減できる構成にする。
- ショップ購入で追加したシールは、既存のシール選択 UI が参照する所持シール一覧へ即時反映する。
- 所持シール 0 件から 1 件目を購入した場合でも、自動では選択状態にせず、一覧からユーザーが選択するまで未選択のままにする。
- すでに別シールを選択中の場合、追加購入後も現在選択を不意に変更しない。
- 所持シールの並び順は「新規購入シールを先頭に追加」とする。

### 6. 所持シール一覧 UI 反映
- ショップ購入後、HUD 左下の所持シール一覧は再描画され、新しいシールが確認できること。
- 所持シール一覧が空から復帰した場合、空表示ラベルは非表示へ切り替える。
- 重複購入した場合は、同じ見た目のシールが複数件一覧に並ぶこと。
- 購入後の所持シールカードは、既存操作と同様にタップで選択できること。

### 7. 実装方式
- UI は既存実装に合わせて UI Toolkit を使用する。
- ショップ一覧のデータソースと所持シールのデータソースは責務を分ける。
- ただし、購入処理ではショップデータから所持データへ `StickerDefinition` を受け渡せる構成にする。
- `OwnedStickerInventorySource` は初期一覧取得専用ではなく、購入時に追加更新できる API を持つ構成にする。
- `HubScreenBinder` はショップ画面の開閉と購入イベントの仲介、および所持一覧再構築の責務を持つ。

## 非機能要件
- 既存のシール配置フェーズ、シールめくりフェーズ、妖精コレクション画面の挙動を壊さない。
- 初期所持 0 件、購入直後、重複購入時のいずれでも null 参照例外を出さない。
- 購入処理は 1 操作ごとに 1 回だけ反映され、重複クリックによる意図しない多重追加が発生しにくい構成にする。
- 購入後の所持一覧更新は、その場で視認できる程度に即時反映されること。
- UI 要素名とメソッド名は、ショップ一覧更新と所持一覧更新の責務差が分かる命名にする。

## 受け入れ条件
- ゲーム開始時、所持シール一覧が 0 件で表示され、例外が発生しない。
- 所持シール 0 件の初期状態でショップ画面を開ける。
- ショップ内の任意のシールを 1 回選択すると、所持シール一覧にそのシールが 1 件追加される。
- 初回購入時、追加されたシールは所持一覧の先頭に表示されるが、自動選択はされない。
- 2 回以上同じシールを選択すると、所持一覧に同じシールが選択回数分だけ追加される。
- 2 回以上同じシールを選択した場合も、新しく購入したシールがその都度所持一覧の先頭へ追加される。
- 既に所持シールを選択済みの状態で別シールを購入しても、現在選択中のシールは維持される。
- 購入時に所持金減算や価格検証は発生しない。
- 購入後もショップ画面は開いたままで、続けて購入操作ができる。
- `閉じる` ボタンまたは背景押下でショップ画面を閉じられる。
- 妖精コレクション画面とショップ画面が同時表示されない。
- 所持シール 0 件でも `Ready` ボタンで既存フェーズ遷移ができる。

## 未確定事項・確認事項
- 購入成功のフィードバックは、今回は Console ログのみで十分とする。
- 重複購入は今回の正式仕様として許可する。
- 将来 1 種 1 枚制限へ変更する可能性に備え、購入判定の責務は分離しやすい設計が望ましい。
