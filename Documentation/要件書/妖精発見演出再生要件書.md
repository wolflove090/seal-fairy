# 妖精発見演出再生要件書

## 背景と目的
- 現在の `PeelSticker3D` では、妖精ありシールをめくり切った際に妖精登録とログ出力を行った後、シールオブジェクトを `Destroy(gameObject, 0.5f)` で短時間後に破棄している。
- 今回は妖精発見時に、シーン上の `ObiRoot` が持つレガシー `Animation` コンポーネントから `discovery` クリップを直接再生し、発見演出をユーザーに視認させたい。
- 発見演出の再生完了を待ってから対象シールを破棄する流れへ変更し、妖精発見体験を明確化する。

## スコープ内
- 妖精発見時にのみ `ObiRoot` の `Animation` コンポーネントで `discovery` をスクリプトから再生する仕様定義
- レガシーアニメーション再生完了待ちと、完了後のシール破棄タイミングの定義
- 既存の妖精登録、ログ出力、シール破棄フローとの接続条件の定義
- 演出再生中の重複起動防止と破棄重複防止の要件整理

## スコープ外
- `discovery` アニメーションクリップ自体の内容変更
- `ObiRoot` の階層構造、Transform、UI レイアウトの変更
- 妖精発見時の SE、追加 UI、報酬、スコア加算の導入
- 妖精なしシールの剥がし演出や破棄タイミングの全面見直し
- Timeline、Animator Controller、Playables への移行

## 機能要件

### 1. 演出トリガー条件
- 妖精発見演出は、シール剥がし完了時に対象シールへ妖精が割り当てられていた場合のみ発生する。
- 妖精なしシールでは `discovery` を再生しない。
- 同一シールに対して、妖精発見演出は 1 回のみ再生する。

### 2. 演出再生対象
- 発見演出の再生対象は、`Assets/Main.unity` 上に存在する `ObiRoot` オブジェクトの `Animation` コンポーネントとする。
- 再生するクリップ名は `discovery` とする。
- 対象クリップはレガシーアニメーションであるため、スクリプトから `Animation` API を用いて直接再生する。
- 実装時は `Animator` や Timeline を介さず、既存の `Animation` コンポーネント構成を維持する。
- `ObiRoot` への参照は名前検索ではなく Inspector から注入する。

### 3. 演出完了待ち
- 妖精ありシールでは、`discovery` の再生開始後、アニメーション完了まで対象シールを破棄しない。
- シール破棄の確定タイミングは、`discovery` の再生完了後とする。
- 演出完了待ちの間、対象シールはシーン上に残り続ける。
- 完了待ちはクリップ長を基準に行い、固定秒数のハードコード待機を前提にしない。
- `ObiRoot` の `Animation` コンポーネントはオート再生を無効化した状態で運用する。

### 4. シール破棄仕様
- 妖精ありシールは、`discovery` 完了後に破棄する。
- 妖精なしシールの破棄タイミングは、既存仕様からの変更有無を実装計画で明記するが、本要件では「発見演出待ちを挟まない」ことを必須条件とする。
- 同一シールに対して破棄処理が複数回走らないこと。
- 破棄遅延により、妖精登録やログ出力の重複が発生しないこと。

### 5. 既存発見処理との整合
- 妖精ありシールでは、既存の妖精登録処理と発見ログ出力を継続する。
- 発見演出追加後も、妖精判定結果とログ出力条件がずれないこと。
- 妖精発見演出の開始条件は、既存の妖精あり判定と同じデータを参照すること。
- 演出再生の追加によって、`StickerRuntimeRegistry` の妖精消費タイミングが不整合にならないこと。
- `discovery` 再生中は、他のシールを新たにめくれないよう入力を抑止する。

### 6. 異常系・フォールバック
- `ObiRoot` が見つからない、または `Animation` コンポーネントを取得できない場合の扱いを実装前に明示する。
- `discovery` クリップが `Animation` に登録されていない場合の扱いを実装前に明示する。
- フォールバック時も、対象シールが破棄されず残留し続ける状態は避ける。

## 非機能要件
- 既存の `PeelSticker3D` の責務を過度に肥大化させず、必要に応じて発見演出再生責務を補助クラスまたは専用コンポーネントへ分離できる構成にする。
- レガシー `Animation` 再生制御は、Unity 実行時のシーン参照切れに配慮した実装にする。
- 既存のシール剥がし操作、妖精登録、妖精コレクション更新の基本挙動を壊さない。
- 演出待ちが加わっても、同時操作で状態不整合が起きにくい単純なライフサイクルを維持する。

## 受け入れ条件
- 妖精ありシールをめくり切ると、`ObiRoot` の `Animation` コンポーネントから `discovery` が再生される。
- 妖精なしシールをめくっても、`discovery` は再生されない。
- 妖精ありシールは、`discovery` の再生完了前に破棄されない。
- 妖精ありシールは、`discovery` の再生完了後に破棄される。
- 妖精発見ログと妖精コレクション登録は従来どおり 1 回のみ実行される。
- 同一シールで発見演出の重複再生、ログ重複、破棄重複が発生しない。
- `ObiRoot` または `discovery` が利用できない異常時でも、シールが永続的に残り続けない。

## 未確定事項・確認事項
- `ObiRoot` の `Animation` をどのコンポーネントが保持し、どの責務境界で `discovery` 再生と完了待ちを管理するかは実装計画で確定する。
- `discovery` 再生失敗時に、ログ出力だけ残して即時破棄するか、既存遅延破棄へフォールバックするかは実装計画で確定する。
