# 妖精コレクション詳細画面要件書

## 背景と目的
- 既存の妖精コレクション画面では一覧表示まで実装済みだが、妖精カード選択後に個別詳細を確認する導線は未実装である。
- ユーザー要望として、妖精カード選択時に詳細モーダルを表示し、妖精ごとの見た目と補足情報を確認できるようにする。
- 添付画像のレイアウト構成を参考にしつつ、既存実装のピンク基調デザインに合わせ、緑を使わない詳細画面仕様を定義する。

## スコープ内
- 妖精コレクション一覧内の妖精カード選択で開く詳細画面の仕様定義
- 詳細画面の表示項目、開閉導線、レイヤー構成、既存一覧画面との関係整理
- 既存の `FairyCatalogSource`、`FairyDefinition`、`FairyCollectionService` を前提にした表示内容の整理
- ピンク基調で統一した詳細画面デザイン要件の定義

## スコープ外
- 妖精コレクション一覧画面そのものの全面リニューアル
- 妖精獲得状態の保存方式変更
- 妖精ごとの新規マスターデータ追加の実装
- 妖精ボイス、アニメーション、演出、サウンドの追加
- フィルタ、並び替え、検索などの一覧機能追加
- 妖精カード以外の HUD 導線変更

## 機能要件

### 1. 画面起動と終了
- 妖精コレクション一覧内の妖精カードを選択すると、同画面上に詳細画面を重ねて表示する。
- 詳細画面はモーダル表示とし、表示中は一覧画面内の他カードや背面 UI を操作できない。
- 詳細画面右上の閉じるボタン押下で画面を閉じる。
- 詳細画面の背景オーバーレイ押下でも画面を閉じる。
- 詳細画面を閉じた後は、妖精コレクション一覧画面の表示状態へ戻る。

### 2. 対象データ
- 詳細画面は一覧で選択された妖精 1 件を対象に表示する。
- 未発見の妖精カードは選択不可とし、詳細画面を開かない。
- 選択対象は `FairyCatalogSource.GetFairies()` から取得した妖精データとする。
- 発見済み判定は `FairyCollectionService.IsDiscovered()` を利用する。
- 詳細画面表示時点で最新の発見状態を参照する。

### 3. 詳細画面レイアウト
- 詳細画面は妖精コレクション一覧の前面中央に重なる単一パネルとする。
- パネル内には少なくとも以下の表示領域を持つ。
- 妖精名
- 妖精画像
- 好きなシール表示
- フレーバーテキスト表示
- 閉じるボタン
- 詳細パネルの外側に背景オーバーレイを配置し、詳細画面外タップで閉じられる構成にする。
- 添付画像の情報構造は踏襲するが、配色は既存実装のピンク系に寄せ、緑系の背景色・枠色は使用しない。

### 4. 表示内容
- 発見済み妖精では `FairyDefinition.DisplayName` と `FairyDefinition.Icon` を表示する。
- 発見済み妖精の好きなシール表示は固定文言 `クール/ワイルド` を表示する。
- 発見済み妖精のフレーバーテキストは、現行データ構造に専用項目がないため、実装時点では固定文言または暫定文言で表示する前提で扱う。
- 妖精画像はパネル中央の専用フレーム内に大きく表示する。
- テキストが複数行になってもパネル外へはみ出さないレイアウトにする。

### 5. 未発見妖精の扱い
- 未発見妖精カードは一覧上で表示を維持するが、選択操作は受け付けない。
- 未発見妖精では妖精名を伏せ表示にする。
- 未発見妖精では画像をシルエット、透明画像、または減色表示のいずれかで通常表示と区別する。
- 未発見妖精の好きなシール表示とフレーバーテキスト表示は、未発見であることが分かる伏せ表示にする。
- 未発見状態でもカードのレイアウト構成は発見済みと同一に保つ。

### 6. デザイン要件
- 詳細画面は既存の妖精コレクション画面と調和するピンク基調の配色に統一する。
- 緑系のベースカラー、差し色、カード背景は使用しない。
- 閉じるボタンは既存 UI と同系統の白ベース角丸ボタンとし、黒系の `X` を視認しやすく表示する。
- 背景オーバーレイは半透明で、背面一覧が見えつつ詳細画面を最前面として認識できる濃度にする。
- 1920x1080 基準で、タイトル・画像・テキストの優先順位が明確に分かる余白設計にする。

### 7. 既存実装との整合
- 実装方式は既存一覧画面と同じく UI Toolkit を前提とする。
- 詳細画面は `FairyCollectionScreen` に内包する追加要素、または詳細専用の UXML / USS として構成できること。
- 開閉制御は既存の `HubScreenBinder` から一元管理できる構成にする。
- 一覧画面を閉じた場合は、開いている詳細画面も同時に閉じる。
- ショップ画面との同時表示は引き続き行わない。

## 非機能要件
- 既存の妖精コレクション一覧、シールショップ、HUD 操作に回帰を出さない。
- 妖精画像未設定、妖精データ未設定、0 件状態でも例外で停止しない。
- テキスト量が増えても詳細パネル内で読み切れる構成にする。
- UI 要素名、USS クラス名、イベント登録箇所は後続の実装で追いやすい命名にする。
- 既存のピンク基調デザインとの一貫性を保ち、詳細画面のみ浮いた色味にしない。

## 受け入れ条件
- 妖精コレクション一覧で妖精カードを選択すると、詳細画面が前面に表示される。
- 詳細画面には妖精名、妖精画像、好きなシール、フレーバーテキスト、閉じるボタンが表示される。
- 詳細画面の配色はピンク基調で、緑系の目立つ色が使用されていない。
- 閉じるボタン押下で詳細画面が閉じる。
- 背景オーバーレイ押下でも詳細画面が閉じる。
- 詳細画面を閉じると妖精コレクション一覧画面へ戻る。
- 未発見妖精カードは一覧上で未発見と判別でき、選択しても詳細画面は開かない。
- 一覧画面を閉じた場合、詳細画面が残留しない。

## 未確定事項・確認事項
- 発見済み妖精のフレーバーテキストは、現時点では固定文言で実装する。
- 未発見妖精カードは選択不可とする。
