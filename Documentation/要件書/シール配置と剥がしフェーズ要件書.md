# シール配置と剥がしフェーズ要件書

## 背景と目的
- 現在のプロジェクトには `PeelSticker3D` による単体シールの剥がし表現があるが、ゲームとしての配置フェーズと剥がしフェーズの往復、および妖精の内包管理は未整備である。
- プレイヤーがシールを配置し、別フェーズでそれを剥がす一連の体験を成立させるため、入力、カメラ移動、配置候補表示、フェーズ遷移、妖精判定までを含む要件を定義する。

## スコープ内
- シール配置フェーズの入力仕様定義
- 画面端マウス移動によるカメラ移動仕様定義
- 配置候補シールの追従表示と残数表示仕様定義
- シール配置時の妖精割り当て仕様定義
- 画面右下ボタンによるフェーズ完了と状態遷移仕様定義
- シール剥がしフェーズの入力仕様定義
- シール剥がし時の消滅処理と妖精発見ログ仕様定義

## スコープ外
- 妖精発見時のスコア計算、報酬、演出拡張
- セーブデータ、リトライ、タイトル画面などのゲーム全体進行管理
- 複数種類のシールや妖精ごとの個体差仕様
- タッチデバイス向けジェスチャー最適化
- サウンド、パーティクル、アニメーションアセットの量産

## 機能要件

### 1. フェーズ管理
- ゲーム状態は少なくとも `SealPlacement` と `SealPeeling` の 2 フェーズを持つ。
- 起動時の初期フェーズは `SealPlacement` とする。
- 画面右下のフェーズ完了ボタン押下で、現在のフェーズに応じて次のフェーズへ遷移する。
- `SealPlacement` 完了時は `SealPeeling` に遷移する。
- `SealPeeling` 完了時は `SealPlacement` に遷移する。
- フェーズ切り替え時、各フェーズで不要な入力は受け付けない。

### 2. シール配置
- プレイヤーが画面をタップまたはクリックした位置に、配置対象シールを 1 枚配置する。
- 配置位置はカメラから見える操作対象平面上の座標へ変換して決定する。
- 配置済みシールは剥がし対象として内部リストで管理する。
- 初期の残り配置数は 10 枚とする。
- 配置操作ごとに残り配置数を 1 減算する。
- 残り配置数が 0 の場合は新規配置を受け付けない。
- 配置済みシール同士の重なりは許可する。

### 3. カメラ移動
- 全フェーズ共通で、マウスカーソルが画面端に入った方向へカメラを移動させる。
- カメラ移動は上下左右に対応する。
- カメラ移動範囲には上限を設け、操作可能エリア外へは移動しない。
- カメラ移動速度、画面端判定幅、移動範囲は調整可能なパラメータとして扱う。

### 4. 配置候補表示
- `SealPlacement` フェーズ中、未配置のシールオブジェクトをカーソル位置に追従表示する。
- 配置候補表示は実際の配置先を示すプレビューとして扱う。
- プレビュー付近に残り配置数を常時表示する。
- `SealPlacement` 以外のフェーズではプレビューと残数表示を非表示にする。

### 5. 妖精の内部管理
- シール配置時に、そのシールへ妖精が潜むかどうかをランダムに決定する。
- 妖精の有無はシールごとに内部データとして保持する。
- 妖精の有無はシールが剥がされるまでプレイヤーへ明示しない。
- ランダム判定に利用する妖精出現確率の初期値は 50% とする。
- 妖精出現確率は調整可能なパラメータとして扱う。

### 6. シール剥がし
- `SealPeeling` フェーズ中、配置済みシールをタップまたはクリックした場合に剥がし演出を再生する。
- 剥がし演出は既存 `PeelSticker3D` の表現を利用または拡張して実現する。
- 剥がし完了後、そのシールオブジェクトはシーンから消滅させ、内部管理リストからも削除する。
- 既に剥がし処理中または削除済みのシールには重複操作を受け付けない。

### 7. 妖精発見ログ
- シール剥がし時、そのシールに妖精が潜んでいた場合はログへ発見メッセージを出力する。
- 妖精がいなかった場合はログを出力しない。
- ログ出力は Unity Console で確認可能な形式とする。

### 8. UI
- UI は UI Toolkit で実装する。
- 画面右下にフェーズ完了ボタンを表示する。
- ボタン表示文言は現在フェーズに応じて変更できる構成とする。
- 残り配置数表示は視認可能な位置とサイズで表示する。
- UI はメインカメラ移動の影響を受けず画面固定で表示する。
- 右下ボタンやログ文言は実装側で妥当な表現を設定する。


## 非機能要件
- 既存の `PeelSticker3D` と競合しない責務分離を行う。
- 既存の `PeelSticker3D` は参考実装として扱い、今回のゲーム進行に必要なシール処理は新規コンポーネントとして再構成する。
- フェーズ管理、入力制御、シールインスタンス管理、UI 表示は責務ごとに分割し、単一コンポーネントへ責務を集中させすぎない。
- 主要なゲームパラメータは Inspector から調整可能にする。
- 単一シーン `Assets/Main.unity` 上で動作確認できる構成にする。
- 将来的なシール種類追加に備え、シール 1 枚分の状態は専用データまたはコンポーネントで保持する。
- WebGL 対応を前提とし、タッチ入力とマウス入力は同一入力経路で扱う。

## 受け入れ条件
- 起動後に `SealPlacement` フェーズが始まり、プレビューシールと残り配置数が表示される。
- クリックまたはタップで見えているエリア内にシールを配置できる。
- 初期状態では 10 枚のシールを配置でき、重なり配置も成立する。
- 画面端にマウスを置くとカメラが移動し、設定された範囲外には出ない。
- 配置した各シールに妖精有無が内部設定される。
- 右下ボタン押下で `SealPeeling` フェーズへ遷移し、配置プレビューが消える。
- `SealPeeling` フェーズ中に配置済みシールを選択すると剥がし演出後にシールが消える。
- 妖精入りシールを剥がした場合、Unity Console に発見ログが出る。
- `SealPeeling` の右下ボタン押下で再び `SealPlacement` フェーズへ戻る。
- `SealPeeling` 完了時に未剥がしシールが残っていた場合、それらは全消去された状態で次の `SealPlacement` フェーズへ戻る。
