# シール配置からシールめくりへのフェーズ遷移要件書

## 背景と目的
- 現状の実装では `TapStickerPlacer` が常時シール配置入力を受け付け、`PeelSticker3D` も常時シールめくり入力を受け付けているため、ゲーム進行としてのフェーズ分離が存在しない。
- 本機能では、ゲーム開始時をシール配置フェーズとし、プレイヤーが画面右上のボタンを押すまでシールを配置できる状態を提供する。
- その後、シールめくりフェーズへ遷移し、配置済みシールに対するめくり操作のみを許可することで、配置とめくりを明確に分離した進行の基盤を作る。

## スコープ内
- 初期フェーズをシール配置フェーズに固定する仕様定義
- フェーズ状態に応じてシール配置入力を有効 / 無効化する仕様定義
- フェーズ状態に応じてシールめくり入力を有効 / 無効化する仕様定義
- 画面右上の UI ボタンによるフェーズ遷移仕様定義
- シールめくりフェーズからシール配置フェーズへ戻る仕様定義
- 既存 UI Toolkit ボタンの繋ぎ込み仕様定義
- フェーズ状態を UI 上で認識できる最低限の表示仕様定義

## スコープ外
- 残り配置数表示、配置プレビュー表示、カメラ移動などの追加 UX 拡張
- 妖精判定、妖精発見ログ、スコア、演出追加などのゲームルール拡張
- セーブ / ロード、リトライ、タイトル画面などの全体進行管理
- 複数種のシールや特殊シールの導入

## 機能要件

### 1. フェーズ状態管理
- ゲーム状態は少なくとも `StickerPlacement` と `StickerPeeling` の 2 フェーズを持つ。
- 起動直後の初期フェーズは `StickerPlacement` とする。
- 現在フェーズは 1 箇所で管理し、入力判定と UI 表示が同じ状態を参照する。

### 2. シール配置フェーズ
- `StickerPlacement` フェーズ中のみ、プレイヤーは画面操作によってシールを配置できる。
- `StickerPlacement` フェーズ中は、シールめくり操作を受け付けない。
- 配置対象や配置位置の算出方法は既存のシール配置実装を踏襲してよい。
- 配置済みシールは、後続のシールめくりフェーズで参照できる形で管理する。

### 3. フェーズ遷移ボタン
- 画面右上にフェーズ遷移用ボタンを常時表示する。
- フェーズ遷移ボタンは既存の UI Toolkit 実装済みボタンを使用し、新規 UI を増設しない。
- `StickerPlacement` フェーズ中にボタンを押下すると、`StickerPeeling` フェーズへ遷移する。
- `StickerPeeling` フェーズ中にボタンを押下すると、`StickerPlacement` フェーズへ遷移する。
- `StickerPeeling` から `StickerPlacement` へ戻る際、未めくりシールは全削除する。
- フェーズ遷移は 1 回のボタン押下に対して 1 回だけ実行される。
- フェーズ遷移後は、入力制御と UI 表示が即時に新フェーズの状態へ切り替わる。

### 4. シールめくりフェーズ
- `StickerPeeling` フェーズ中のみ、配置済みシールに対するめくり操作を受け付ける。
- `StickerPeeling` フェーズ中は、新しいシール配置を受け付けない。
- シールめくり対象は、`StickerPlacement` フェーズで配置済みのシールに限定する。
- 既にめくり中、またはめくり済みのシールへの重複入力は受け付けない。

### 5. UI 表示
- UI は既存構成に合わせて UI Toolkit で実装する。
- 画面右上ボタンは既存実装を流用し、本対応ではフェーズ制御との繋ぎ込みを担当する。
- 画面右上ボタンは現在フェーズに応じて文言を切り替える。
- プレイヤーが現在のフェーズを認識できるよう、フェーズ表示はボタン文言のみで表現する。
- UI はワールド空間のシール操作とは独立した画面固定表示とする。

## 非機能要件
- フェーズ判定は `TapStickerPlacer` と `PeelSticker3D` に直接分散させず、共通の状態参照元を設ける。
- 既存のシール配置処理とシールめくり処理は可能な限り再利用しつつ、入力の有効 / 無効のみをフェーズで切り替えられる構成にする。
- フェーズ遷移の責務と UI 更新の責務を分離し、今後のフェーズ追加に耐えられる構成にする。
- `Assets/Main.unity` 上で単一シーンとして動作確認できること。

## 受け入れ条件
- アプリ起動直後は `StickerPlacement` フェーズで開始する。
- 起動直後の状態ではシール配置ができ、シールめくりはできない。
- 画面右上にフェーズ遷移ボタンが表示されている。
- 配置フェーズ中に右上ボタンを押すと `StickerPeeling` フェーズへ遷移する。
- フェーズ遷移後は新規シール配置ができない。
- フェーズ遷移後は配置済みシールのみをめくれる。
- めくりフェーズ中に未配置領域をクリック / タップしても新規シールは生成されない。
- めくりフェーズ中に同じシールへ重複入力しても処理が破綻しない。
- めくりフェーズ中に右上ボタンを押すと `StickerPlacement` フェーズへ戻る。
- `StickerPeeling` から `StickerPlacement` へ戻る際、未めくりシールはシーン上から全削除される。
- フェーズ遷移のたびに右上ボタン文言が現在フェーズに対応した内容へ更新される。

## 未確定事項・確認事項
- ボタン文言の具体的な文言は設計で確定する。少なくとも現在フェーズと次の操作意図が分かる表現にする。
