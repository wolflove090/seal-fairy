# 妖精ごとの好きなシールとフレーバー個別設定要件書

## 背景と目的
- 現在の妖精詳細 UI では、好きなシールとフレーバーテキストが `HubScreenBinder` の固定文言で表示されており、全妖精で同じ内容になっている。
- 妖精ごとの個性を UI 上で表現できるようにするため、妖精ごとに好きなシールとフレーバーを個別設定し、妖精コレクション詳細画面に反映できるようにする。
- 既存の妖精カタログ運用と UI Toolkit ベースの表示構成を維持しつつ、Inspector から設定可能なデータ構造へ整理する。

## スコープ内
- 妖精ごとの好きなシール表示文言の個別設定仕様定義
- 妖精ごとのフレーバーテキストの個別設定仕様定義
- `FairyCatalogSource` が保持する妖精マスタに必要なデータ項目整理
- 妖精コレクション詳細画面と一覧カードへの表示反映要件整理
- 未設定時のフォールバック表示仕様定義

## スコープ外
- 妖精コレクション画面のレイアウト全面改修
- 妖精発見条件、発見状態保存、抽選ロジックの変更
- 新規妖精アセットの追加
- 多言語対応、改行自動調整、文字装飾強化
- シールショップや配置フェーズ UI の仕様変更

## 機能要件

### 1. 妖精マスターデータ
- 各 `FairyDefinition` は、既存の `id`、`displayName`、`icon` に加えて、好きなシール表示用の自由入力文字列を 1 件保持できる。
- 各 `FairyDefinition` は、フレーバーテキストを個別に保持できる。
- これらの項目は Unity Inspector 上で妖精ごとに設定できる。
- 妖精マスタ未設定や要素欠落があっても、参照時に例外で停止しない。

### 2. 妖精詳細画面への反映
- 妖精コレクション詳細画面を開いたとき、選択した妖精の設定値を表示する。
- 好きなシール欄には、選択した妖精に設定された好きなシール表示用データを表示する。
- フレーバーテキスト欄には、選択した妖精に設定されたフレーバーテキストを表示する。
- 既存の固定文言定数は、個別設定値を優先して参照する構成へ置き換える。

### 3. 妖精一覧カードへの反映
- 妖精コレクション一覧カードの「好きなシール」欄も、発見済み妖精については個別設定値を表示する。
- 未発見妖精カードは、従来どおり伏せ表示を維持し、個別設定値を露出しない。
- 一覧カードで表示する好きなシール文言は、詳細画面と同じ元データを参照する。

### 4. 未設定時のフォールバック
- 好きなシール表示用データが未設定の場合は、UI 上で空欄にならない代替文言 `*****` を表示する。
- フレーバーテキストが未設定の場合も、UI 上で空欄にならない代替文言 `*****` を表示する。
- フォールバック文言は全妖精共通で `*****` に統一し、実装箇所を 1 か所に寄せて管理できる構成とする。

### 5. 表示ルール
- 既存の発見済み/未発見判定ロジックは変更しない。
- 発見済み妖精では、名前、画像、好きなシール、フレーバーの各表示が同一妖精データを元に揃う。
- フレーバーテキストは複数行表示を許容し、既存詳細パネル内で読み切れる範囲に収める。

## 非機能要件
- 既存の妖精コレクション開閉、妖精カード選択、ショップ UI の挙動に回帰を出さない。
- データ追加に伴っても `HubScreenBinder` に表示文言の責務を持たせすぎず、表示値の取得元が追いやすい構成にする。
- 新しい項目を 1 件追加するだけで妖精ごとの差分を設定できるよう、運用コストを増やしすぎない。
- null、空文字、改行入り文字列を扱っても描画が破綻しない。

## 受け入れ条件
- Inspector 上で妖精ごとに異なる好きなシールとフレーバーを設定できる。
- 発見済み妖精 A と妖精 B を詳細画面で開いたとき、それぞれ異なる表示内容になる。
- 妖精コレクション一覧の発見済みカードでも、妖精ごとに異なる好きなシール表示が確認できる。
- 未発見妖精カードでは、好きなシールやフレーバーの個別内容が見えない。
- 設定未入力の妖精がいても、詳細画面表示時にエラーにならず代替文言が出る。

## 未確定事項・確認事項
- 好きなシールは `StickerDefinition` 参照ではなく、妖精ごとの自由入力文字列として管理する。
- 一覧カードに表示するのは「好きなシール」のみとし、フレーバーは詳細画面専用表示のままにする想定で整理している。
- 好きなシールとフレーバーの未設定時フォールバック文言は、一律 `*****` とする。
