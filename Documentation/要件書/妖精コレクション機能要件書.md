# 妖精コレクション機能要件書

## 背景と目的
- 既存 HUD には `妖精` ボタンが存在するが、押下後に表示するコレクション画面は未実装である。
- 既存実装では妖精マスターデータ、発見状態管理、発見ログまでは存在するため、これらを利用して「獲得済み妖精を一覧で確認できる UI」を追加したい。
- 添付ワイヤーに沿って、プレイヤーが発見済み・未発見の妖精をひと目で判別できる妖精コレクション画面を定義する。

## スコープ内
- HUD の `妖精` ボタン押下で開くコレクション画面の仕様定義
- 妖精マスターデータと発見状態を使った一覧表示仕様の定義
- 未獲得妖精のグレーアウトおよび未発見表示仕様の定義
- 一覧のスクロール表示、発見数表示、閉じる操作の仕様定義
- ワイヤー準拠の UI 構成と表示項目の定義

## スコープ外
- 妖精獲得状態の永続保存方式追加
- 妖精詳細画面への遷移
- 妖精一覧画面内での並び替え、フィルタ、検索
- 妖精発見演出やサウンドの追加変更
- ショップ画面の実装変更
- 妖精ごとのレアリティ表示や追加パラメータ表示

## 機能要件

### 1. 画面起動と終了
- HUD 右下の `妖精` ボタン押下で妖精コレクション画面を表示する。
- 妖精コレクション画面の表示中は、一覧パネルと閉じるボタンが前面に出る。
- コレクション画面内の `閉じる` ボタン押下で妖精コレクション画面を閉じる。
- 背景オーバーレイ押下でも妖精コレクション画面を閉じる。
- コレクション画面を閉じた後は、元の HUD 表示状態へ戻る。

### 2. 一覧表示対象
- 一覧には `FairyCatalogSource` に登録されている全妖精を表示する。
- 表示順は `FairyCatalogSource` に登録された順を初期仕様とする。
- コレクション画面を開いた時点で、最新の獲得状態を用いて各妖精の表示状態を決定する。
- 妖精が 0 件のときは、一覧領域に空表示メッセージを出し、発見数は `0/0` とする。

### 3. 一覧レイアウト
- 一覧はワイヤーどおり右側の大型パネルとして表示する。
- パネル上部にタイトル `妖精一覧` を表示する。
- 一覧領域は縦スクロール可能にする。
- 一覧アイテムは 2 列グリッドを基本とし、画面幅不足時の崩れを防げる構成にする。
- 各アイテムカードには少なくとも以下を含める。
- 妖精名
- 妖精画像
- 好きなシールを表示する補足テキスト領域
- 下部には `閉じる` ボタンと `発見した数: X/Y` 表示を配置する。

### 4. 獲得済み・未獲得表示
- 獲得済み妖精は通常表示する。
- 未獲得妖精は妖精画像部分をグレーアウト表示する。
- 未獲得妖精の補足テキストには `未発見` を表示する。
- 未獲得妖精の名前は `*****` で伏せ字表示にする。
- グレーアウト時も「未獲得であること」は視覚的に明確であること。
- 未獲得妖精も一覧には残し、枠自体は非表示にしない。
- 発見数 `X` は獲得済み妖精数、総数 `Y` は一覧対象妖精総数とする。

### 5. 各妖精カードの表示内容
- 獲得済み・未獲得を問わず、少なくとも妖精名と画像枠をカード上に持つ。
- 妖精画像は `FairyDefinition.Icon` を使用する。
- 獲得済み妖精名は `FairyDefinition.DisplayName` を使用する。
- ワイヤーにある補足文言領域はカード下部に確保する。
- 好きなシール文言は固定文言 `好きなシール: ポップ・小さい` を表示する。
- 将来的には妖精ごとの個別データへ置き換える前提とするが、今回の実装では追加データを考慮しない。

### 6. UI 構成
- 実装方式は既存 HUD と同じく UI Toolkit を前提とする。
- コレクション画面は既存 HUD と分割した新規 UXML / USS で実装する。
- `HubScreenBinder` から表示制御できる構成にする。
- コレクション画面表示用のルート要素は、通常 HUD の上に重ねるオーバーレイとして扱う。
- 背景は画面全体を覆う半透明グレーのオーバーレイにする。
- 背景オーバーレイは背面ゲーム操作と背面 HUD 操作を遮断する。
- オーバーレイ表示中も金額表示や他 HUD 要素のレイアウトが崩れないこと。

### 7. 状態取得
- 獲得済み判定は `FairyCollectionService` が持つ発見状態を利用する。
- 一覧表示用に「全妖精一覧」「指定妖精が獲得済みか」「獲得済み総数」を取得できる窓口を用意する。
- UI 側が獲得済み ID 集合の内部実装を直接知らない構成にする。

## 非機能要件
- 既存のシール配置フェーズ、シールめくりフェーズの操作を壊さない。
- 妖精アイコン未設定や妖精データ 0 件でも例外で停止しない。
- 一覧項目数が増えても、画面外へはみ出さずスクロールで閲覧できる。
- UI 要素名、クラス名、バインド箇所は後続の実装で追いやすい命名にする。
- ワイヤー基準解像度は 1920x1080 とし、同等の 16:9 画面で大きくレイアウト崩れしないこと。

## 受け入れ条件
- ゲーム画面で `妖精` ボタンを押すと、妖精コレクション画面が表示される。
- コレクション画面に登録済み全妖精のカードが表示される。
- 一覧項目がパネル高さを超える場合でも、スクロールして最後まで確認できる。
- 未獲得妖精は画像部分のグレーアウトと `未発見` 表示で獲得済み妖精と明確に区別できる。
- 未獲得妖精名は `*****` の伏せ字表示になる。
- 画面下部に `発見した数: X/Y` が表示され、X は獲得済み数、Y は全妖精数と一致する。
- `閉じる` ボタン、または背景オーバーレイ押下でコレクション画面が閉じる。
- 妖精データ 0 件でも画面は開けて、空表示のまま正常動作する。
- コレクション画面表示中は背面ゲーム操作と HUD 操作が発生しない。
