# ショップUIと周辺UIブラッシュアップ要件書

## 背景と目的
- 現在の HUD とシールショップ UI は機能面では成立しているが、レイアウト、配色、装飾、情報密度が添付モックの画面と大きく異なる。
- 今回は既存の購入機能、所持金表示、所持シール一覧表示を維持したまま、ショップ UI と周辺 HUD をモック準拠の見た目へ刷新する。
- 画面全体の印象を「玩具感のあるポップなピンク基調 UI」に寄せ、プレイヤーが所持金、所持シール一覧、ショップ一覧を視覚的に区別しやすい状態を作る。

## スコープ内
- HUD 左上の所持金パネルを添付モックに寄せた見た目へ刷新する仕様
- HUD 左下の所持シール一覧パネルを添付モックに寄せた見た目へ刷新する仕様
- 右側ショップオーバーレイ全体の構造、色、余白、装飾、カードデザインを添付モック準拠へ更新する仕様
- ショップカード内の画像、名称、価格表示レイアウトを添付モック準拠へ更新する仕様
- 閉じるボタン、スクロール領域、背景オーバーレイの見た目を更新する仕様
- 現行の UI Toolkit / `HubScreenBinder` 構成を前提とした UI 命名と責務整理

## スコープ外
- ショップ購入ロジック、所持金計算ロジック、所持シール更新ロジックの仕様変更
- 新規演出、アニメーション、SE 追加
- 妖精コレクション画面のデザイン刷新
- 3D 背景、床、ステージオブジェクトの見た目変更
- セーブデータ、課金、バックエンド連携
- ショップのカテゴリ切り替え、ソート、フィルタ、購入確認ダイアログ

## 機能要件

### 1. 全体レイアウト
- ショップを開いていない通常時は、HUD 左上に所持金パネル、左下に所持シール一覧パネルを表示する。
- ショップ表示時は、画面右側に縦長の大型ショップパネルをオーバーレイ表示する。
- ショップパネルは画面の右端に寄せて表示し、左側には半透明の背景オーバーレイを敷く。
- ショップ表示中も、左側のゲーム背景と HUD 左上 / 左下パネルは視認可能な状態を維持する。
- 右下の `ショップ` / `妖精` ボタン群は、ショップ表示中も明示的には非表示にしない。結果としてショップパネルの下に隠れる見え方は許容する。
- 現行どおり、ショップ画面と妖精コレクション画面は同時表示しない。

### 2. 所持金パネル
- 所持金パネルは HUD 左上に固定表示する。
- パネルは角丸矩形とし、ベースカラーはモック同様の明るいピンク系とする。
- パネル左側にコインアイコン、右側に所持金数値を大きく表示する。
- 表示文言は現行の `お金：1000円` 形式ではなく、モック同様に数値主体の表示とする。
- 所持金数値は 6 桁以上でも崩れにくいよう、右側余白と最小幅を確保する。
- コインアイコンは新規アセットを追加して対応する。
- 通貨更新時は現行どおり即時反映する。

### 3. 所持シール一覧パネル
- 所持シール一覧パネルは HUD 左下に固定表示する。
- パネル上部にピンク系ヘッダー帯を配置し、`シール一覧` タイトルを大きく表示する。
- パネル本体は薄いピンク系のコンテンツ領域とし、内部にスクロール可能な一覧領域を持つ。
- 一覧カードはモック同様に小型の角丸タイルを複数列で並べる。
- 各所持シールタイルには少なくともシール画像を表示する。
- シール名表示はモック同様にタイル下部またはタイル内下辺に小さく収める構成とする。
- 既存の選択状態は視認できる見た目で維持する。ただし配色と枠表現は新デザインに合わせて再設計してよい。
- 所持シール 0 件時は空表示メッセージを表示し、デザイン刷新後も例外を出さない。

### 4. ショップオーバーレイ
- ショップオーバーレイは右側に大きなピンク系パネルとして表示する。
- パネル上部にタイトル帯を設け、`シールショップ` を大きく表示する。
- タイトル帯の右上に大型の閉じるボタンを配置する。
- 閉じるボタンはモック同様に白地の角丸矩形上に `X` を表示する構成とする。
- ショップ一覧コンテンツ領域はタイトル帯の下に配置し、薄いピンクの背景と内側余白を持つ。
- コンテンツ領域はスクロール可能とし、縦スクロール時もヘッダーと閉じるボタンは固定表示する。

### 5. ショップカード
- ショップカードは 1 行 3 列を基本とするグリッド配置とする。
- カード上部は黄緑系の画像領域とし、その中にシール画像を大きく表示する。
- 画像領域下部にシール名を白文字で大きく表示する。
- カード下部に白地の価格プレートを別パーツとして配置し、価格数値をピンク文字で中央表示する。
- 価格表示はモック同様に `500` のような数値主体とし、`円` は表示しない。
- シール名が長い場合も 2 行以内で収まり、カード高さが大きく崩れないようにする。
- 画像未設定時はプレースホルダー色面のみでもレイアウトを崩さない。

### 6. 購入可能 / 不可状態の見た目
- 購入可能カードは通常彩度で表示する。
- 購入不可カードは現行仕様どおり操作不可としつつ、新デザインを保ったまま視覚的に無効状態と分かる表現にする。
- 無効状態ではカード全体を単純に灰色一色へ潰さず、画像、名称、価格の各要素に一貫した減衰表現を適用する。
- 購入可能 / 不可の切り替えは所持金変化時に即時反映する。

### 7. 既存ロジックとの整合
- `HubScreenBinder` が保持しているショップ開閉、一覧再構築、所持金反映、購入イベント仲介の責務は維持する。
- `StickerShopCatalogSource`、`OwnedStickerInventorySource`、`CurrencyBalanceSource` の API 仕様は原則変更しない。
- UI ブラッシュアップのために UXML 要素の階層や要素名を変更する場合は、Binder 側参照名も同期して更新する。
- ショップ一覧 0 件時、所持シール 0 件時、画像未設定時、価格 0 の時でも UI が破綻しない。

### 8. 画面サイズ追従
- 現在の想定解像度は添付モック相当の 1920x1080 を基準とする。
- 16:9 の前後で解像度が変わっても、主要パネルが画面外にはみ出しにくい構成とする。
- ショップパネルは高さ方向にフル表示を維持しつつ、内部スクロールで一覧量増加に対応する。
- 左下所持シール一覧は、一定件数を超えた時点でスクロールし、パネル外にはみ出さない。

## 非機能要件
- UI 実装は既存方針どおり UI Toolkit を使用する。
- USS は役割単位でクラス名を整理し、色、角丸、余白、フォントサイズを把握しやすい命名にする。
- モック追従のための定数値は USS に集約し、`HubScreenBinder` にレイアウト値を埋め込まない。
- 既存機能を壊さないことを優先し、ロジック改修は必要最小限に留める。
- 表示更新時に null 参照例外や、ショップ開閉を跨いだ要素参照切れを発生させない。
- フォントは現時点でプロジェクト内に存在する使用可能資産からモックに近いものを採用し、差し替えしやすい構成にする。
- 長いシール名、所持金大値、販売数増加時でも可読性を大きく損なわない。

## 受け入れ条件
- 通常 HUD 表示時、左上の所持金パネルがピンク基調、コインアイコン付き、数値主体の見た目で表示される。
- 通常 HUD 表示時、左下の所持シール一覧がピンクヘッダー付きパネルで表示される。
- ショップを開くと、画面右側に大型のピンク系ショップパネルが表示される。
- ショップパネル上部に `シールショップ` タイトルと大型 `X` 閉じるボタンが表示される。
- ショップ一覧は 3 列グリッドで表示され、各カードに画像、シール名、価格プレートが表示される。
- ショップカードの配色と情報配置が添付モックに近い見た目になっている。
- 購入可能カードと購入不可カードが視覚的に判別できる。
- 所持金更新後、HUD 左上表示とショップ内表示が同一値で即時更新される。
- 所持シール購入後、左下の所持シール一覧が新デザインのまま更新される。
- ショップ開閉、購入、所持シール選択、フェーズ切り替えの既存動作が維持される。
