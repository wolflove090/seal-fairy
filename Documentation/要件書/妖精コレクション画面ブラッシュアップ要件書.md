# 妖精コレクション画面ブラッシュアップ要件書

## 背景と目的
- 既存の妖精コレクション画面は `HudScreenBinder`、`FairyCollectionScreen.uxml`、`FairyCollectionScreen.uss` により基本機能は実装済みだが、レイアウトと意匠が添付画像の方向性と一致していない。
- 今回は既存の発見状態連携、開閉導線、カード生成ロジックを活かしつつ、妖精コレクション画面を添付画像ベースの見た目へブラッシュアップする。

## スコープ内
- `妖精` ボタン押下で開くコレクション画面のレイアウト再設計
- 添付画像を基準にしたヘッダー、閉じるボタン、内側フレーム、カード見た目の再定義
- 既存の妖精カード生成ロジックに対する表示項目と状態別見た目の再定義
- 既存 HUD と同時表示されたときの重なり順、表示範囲、操作遮断方式の再定義
- 既存の `FairyCatalogSource` と `FairyCollectionService` を利用した表示更新仕様の整理

## スコープ外
- 妖精獲得状態の保存方式変更
- 妖精詳細画面、フィルタ、ソート、検索の追加
- 妖精ごとの新規マスターデータ追加
- 妖精ボタン以外の HUD レイアウト変更
- シールショップ画面のブラッシュアップ
- 発見演出、サウンド、アニメーションの追加

## 機能要件

### 1. 基本動作
- HUD の `妖精` ボタン押下で妖精コレクション画面を開く。
- 妖精コレクション画面表示中はシールショップ画面を閉じ、ショップ画面と同時表示しない。
- 妖精コレクション画面は `X` ボタン押下で閉じる。
- 既存の開閉制御は `HubScreenBinder` から行い、表示データは画面を開くたびに再取得する。
- 妖精一覧対象は `FairyCatalogSource` に登録された全件とする。

### 2. 画面全体レイアウト
- 画面は全画面オーバーレイの構造を維持しつつ、見た目は「右側固定の大型パネル」が前面に出る構成にする。
- 左側のゲーム背景、所持金パネル、シール一覧は添付画像のように視認できる状態を維持する。
- 妖精コレクション画面表示中は、左側が見えていても背面ゲーム操作と背面 HUD 操作は受け付けない。
- 右側パネルは画面の上端から下端まで伸びる縦長レイアウトとし、1920x1080 基準で横幅が広すぎて左側 HUD を圧迫しない寸法にする。
- パネル外周は濃いピンク、内側表示領域は淡いピンク系で二重フレーム構造にする。

### 3. ヘッダーと閉じるボタン
- パネル上部に大きなタイトル帯を配置し、タイトル文言は `妖精コレクション` とする。
- タイトル帯は濃いピンク地に白文字を基本とし、添付画像に近い強いコントラストを持たせる。
- 閉じるボタンは白ベースの角丸四角ボタンとし、右上に大きく配置する。
- 閉じるボタンの見た目は既存 `StickerShopScreen` のクローズボタンと同系統の立体表現を持たせる。
- 閉じるボタンには `X` の視認性が高い黒系表示を用いる。

### 4. 一覧領域
- 一覧領域はヘッダー下の内側フレーム内に配置する。
- 一覧領域は縦スクロールを維持し、件数増加時も最下部まで確認できる。
- 1920x1080 基準では 3 列グリッドでカードを配置する。
- 画面幅が不足する場合は 2 列まで縮退可能な構成にし、カード同士の重なりやパネル外はみ出しを防ぐ。
- スクロールバーは一覧領域の右端に収まり、カードと視覚的に干渉しない。
- データ 0 件時は内側フレーム内に空表示メッセージを出し、スクロール不能でも破綻しない。

### 5. 妖精カードの見た目
- カードは添付画像に合わせ、黄緑系の外装と丸みのある矩形で構成する。
- カード上部に妖精名を大きく表示する。
- カード中央に淡いクリーム色の画像フレームを持たせ、その中に妖精アイコンを中央配置する。
- カード下部に `好きなシール：` ラベルと値表示領域を 2 行構成で配置する。
- カード全体は現状実装より一回り大きくし、3 列時でもタイトル・画像・補足文が詰まりすぎない余白を確保する。
- カードはタップやクリック対象にはせず、今回は一覧表示専用とする。

### 6. 発見済み・未発見の表示
- 発見済み妖精は `FairyDefinition.DisplayName` と `FairyDefinition.Icon` を通常表示する。
- 未発見妖精もカード枠は表示したまま残す。
- 未発見妖精は名前、画像、補足文を既存仕様に準じて伏せ表示にする。
- 未発見妖精名は `？？？` 表記を採用する。
- 未発見状態でも「同じカードデザインの中で未発見である」と判別できるよう、グレースケール化、半透明化、伏せ字のいずれかを組み合わせる。
- 発見済みと未発見でカードサイズやグリッド位置は変えない。

### 7. 表示文言
- タイトル文言は `妖精コレクション` に統一する。
- 補足文言の 1 行目は `好きなシール：` とする。
- 補足文言の 2 行目は固定文言 `クール/ワイルド` を表示する。
- 妖精ごとに好みのシールを出し分ける仕組みは今回追加しない。
- 発見数表示 `X/Y` は画面右下に配置する。

### 8. 既存実装との整合
- 既存の `OpenFairyCollection()`、`CloseFairyCollection()`、`RefreshFairyCollection()` の責務は維持する。
- 既存の `CreateFairyCard()` 相当の組み立て処理は、必要に応じて子要素構造を増やしてもよい。
- 妖精一覧データは `FairyCatalogSource.GetFairies()` を使い続ける。
- 発見済み判定は `FairyCollectionService.IsDiscovered()` と `GetDiscoveredCount()` を使い続ける。
- `FairyDefinition` に新規 serialized field を追加しなくても成立する実装を優先する。

## 非機能要件
- UI Toolkit ベースの実装を維持する。
- 既存の HUD 表示、フェーズ切替、ショップ導線に回帰を出さない。
- 1920x1080 を基準に、少なくとも同等の 16:9 解像度で大きく崩れない。
- カード数増加時もフレーム、スクロール、余白が破綻しない。
- アイコン未設定や妖精 0 件でも例外で停止しない。
- 変更対象の UXML 名、USS クラス名、C# 参照名は後続実装で追いやすい命名にする。

## 受け入れ条件
- `妖精` ボタン押下で、添付画像に近い右側固定の妖精コレクションパネルが表示される。
- タイトルが `妖精コレクション` に変わり、右上に白ベースの `X` ボタンが表示される。
- 左側の所持金パネルとシール一覧は表示されたまま、右側パネルが前面に重なる。
- 一覧が 1920x1080 基準で 3 列に並び、件数超過時は縦スクロールで全件確認できる。
- 各カードが黄緑系の外装、中央画像フレーム、下部補足文の構成になっている。
- 発見済み妖精は名前と画像が通常表示され、未発見妖精は同じレイアウト内で伏せ表示される。
- 右下に発見数 `X/Y` が表示される。
- 画面表示中にショップ画面は同時表示されない。
- `X` ボタンで閉じたとき、背面 HUD のレイアウトは崩れず元の操作状態へ戻る。
