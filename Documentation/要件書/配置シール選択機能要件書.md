# 配置シール選択機能要件書

## 背景と目的
- 現在の `TapStickerPlacer` は単一の `templateSticker` を複製して配置しており、プレイヤーが配置対象のシールを切り替える手段がない。
- シール配置フェーズ中に所持シール一覧から任意のシールを選択できるようにし、選択中シールをそのまま配置対象へ反映することで、複数種類のシールを使った遊びを成立させる。
- UI は添付ワイヤーに沿って画面左下に固定表示し、既存 HUD と干渉しない形で追加する。

## スコープ内
- シール配置フェーズ用 HUD に左下固定の所持シール一覧パネルを追加する。
- 所持シール一覧をスクロール可能な UI として表示する。
- 一覧内のシールを選択して、現在の配置対象シールを切り替えられるようにする。
- 選択中シールを配置プレビューと実配置の両方へ反映する。
- 初期選択状態、選択不可時の扱い、フェーズ切り替え時の表示ルールを定義する。

## スコープ外
- 所持シールの購入、解放、消費などのショップ機能
- シールごとの価格、レアリティ、能力差分
- シール一覧セルの凝った演出、アニメーション、フィルタ、並び替え
- セーブデータへの選択状態保存
- シールごとの残数管理

## 機能要件

### 1. 対象フェーズ
- 本機能は `SealPlacement` フェーズで使用する。
- `SealPeeling` フェーズ中は所持シール一覧を非表示にする。
- フェーズが `SealPlacement` に戻った際は、シール選択状態を解除する。
- シールが未選択の状態では、画面をクリックまたはタップしてもシールの配置は行わない。

### 2. 所持シール一覧表示
- 画面左下に、ワイヤー準拠の矩形パネルとして所持シール一覧を表示する。
- 一覧は UI Toolkit のスクロール可能なコンテナで構成する。
- 一覧にはプレイヤーが現在所持している配置可能シールを全件表示する。
- 一覧の各要素には最低限、シールを識別できる見た目を表示する。
- 一覧セルの表示内容は画像を用いる。
- 一覧の表示件数がパネル高さを超える場合、スクロールして全件確認できること。
- 所持シールが 0 件の場合でも UI は破綻させず、空状態が判別できる表示にする。

### 3. シール選択
- 一覧内のシール要素はクリックまたはタップで選択できる。
- 選択されたシールは一覧内で視覚的に判別できる状態にする。
- 同時に選択できるシールは 1 件のみとする。
- 既に選択中のシールを再度押下しても未選択状態には戻さず、選択状態を維持する。
- 選択対象が変更された時点で、以降の配置対象シールを新しい選択内容へ切り替える。

### 4. 配置対象反映
- 現在の配置プレビューは、選択中シールの見た目と同一のものを表示する。
- クリックまたはタップで配置されるシールは、選択中シールと同じ prefab / 見た目 / 向き / 基本設定を引き継ぐ。
- シール選択を変更しても、既に配置済みのシールは変更しない。
- 妖精の内包判定や配置済みシール管理は、シール種類に依存せず既存ルールを維持する。

### 5. データ管理
- 所持シール一覧は、配置可能なシール定義のコレクションから構築できるようにする。
- 所持シール一覧のデータ供給元は、今後の所持品管理データに接続する。
- 各シール定義は少なくとも、一覧表示に必要な識別情報と、配置時に生成する prefab 参照を持つ。
- `TapStickerPlacer` 相当の配置処理は、固定の `templateSticker` 1 つではなく「現在選択中のシール定義」を参照して配置を行う。
- シール選択 UI の初期選択はシール一覧の先頭要素とする。
- `SealPlacement` フェーズへ再入時は初期選択を引き継がず、未選択状態から開始する。

### 6. UI レイアウト
- 既存の左上所持金、右上準備完了、右下妖精・ショップボタンと同時表示しても重ならないこと。
- 所持シール一覧パネルは画面左下固定とし、カメラ移動の影響を受けないこと。
- 1920x1080 を基準に、ワイヤー相当の十分なサイズを確保する。
- 解像度やアスペクト比が変わっても、左下基準の配置が大きく崩れないこと。

## 非機能要件
- 既存の UI Toolkit ベース HUD 構成へ追加し、既存要素の責務を不必要に混在させない。
- シール定義の追加時に UI と配置処理の両方へ反映しやすいデータ構造にする。
- 主要パラメータは Inspector から差し替え可能にする。
- マウス入力とタッチ入力で同一の選択操作を行えること。
- UI 選択操作とワールド上のシール配置操作が競合しないようにする。

## 受け入れ条件
- `SealPlacement` フェーズ中、画面左下に所持シール一覧のスクロール UI が表示される。
- 所持シールが複数ある場合、スクロール内の任意のシールをクリックまたはタップで選択できる。
- 選択したシールに応じて、配置プレビューの見た目が即時に切り替わる。
- 選択後に画面をクリックまたはタップすると、選択したシールが配置される。
- シール選択を変更しても、既に配置済みのシールはそのまま残る。
- `SealPeeling` フェーズへ遷移すると、所持シール一覧 UI は非表示になる。
- 再び `SealPlacement` フェーズへ戻ると、シールは未選択状態で表示され、選択するまで配置できない。
- UI 操作中に意図せずワールドへシールが配置されない。
