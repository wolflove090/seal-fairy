# シール妖精配置と剥がしログ更新要件書

## 背景と目的
- 現在のプロジェクトでは、シール配置時に妖精を内部データとして保持し、シール剥がし時にその結果を参照する一連の仕様が未接続、またはユーザー実装前提の状態である。
- 今回はユーザー自身が実装を行う前提で、作業対象と確認ポイントを限定した要件を明文化し、実装着手時の判断ぶれをなくす。

## スコープ内
- シールを 1 枚配置した時点で、そのシールに妖精がいるかどうかを 50% の確率で決定する。
- 妖精有無の判定結果を、各シールの内部状態として保持する。
- シールをめくった時に、対象シールに妖精がいた場合のみログを出力する。
- 妖精がいなかった場合は、妖精発見ログを出力しない。
- ユーザーが実装時に確認すべき作業対象、確認手順、完了条件を定義する。

## スコープ外
- 妖精発見時の報酬、スコア加算、UI 表示追加
- 妖精出現率をゲーム中に変更する UI
- 妖精発見時の追加演出、SE、パーティクル
- シールの見た目や剥がしアニメーション自体の刷新
- 配置可能枚数、カメラ移動、フェーズ全体仕様の再定義

## 機能要件

### 1. シール配置時の妖精判定
- シール配置確定ごとに、対象シールへ妖精有無を 1 回だけ割り当てる。
- 妖精判定の初期確率は 50% とする。
- 同じシールに対して、配置後に妖精判定をやり直さない。
- 妖精判定結果は、シールを識別できる単位で保持する。

### 2. シール状態への保持
- シールごとの内部状態に、少なくとも「妖精がいるかどうか」を保持できること。
- めくり処理側は、対象シールの内部状態から妖精有無を参照できること。
- 配置時に設定した妖精有無は、めくるまで失われないこと。

### 3. シールめくり時のログ出力
- シールをめくる処理が完了した時点で、対象シールの妖精有無を判定する。
- 妖精がいた場合のみ、Unity Console で確認できるログを 1 回出力する。
- 妖精がいなかった場合は、妖精発見に関するログを出力しない。
- 同一シールに対して重複してログを出力しない。

### 4. ログ文言
- ログ文言は、対象シールに妖精がいたことが判別できる内容にする。
- 文言の細部は実装側で決めてよいが、少なくとも「妖精がいた」事実が分かること。

## 非機能要件
- 妖精判定ロジックとログ出力ロジックは、可能な限り分離して責務を明確にする。
- 妖精有無の保持先は `MonoBehaviour` の一時状態ではなく、既存のシール管理構造に自然に統合する。
- 既存のシール配置処理、シールめくり処理の基本挙動を壊さない。
- 確率判定を追加しても、手動確認で挙動差分を追える程度の単純な構成にする。
- 今回の乱数取得は `UnityEngine.Random` を直接参照する実装でよい。

## 受け入れ条件
- シールを複数回配置した時、各シールごとに独立して妖精有無が保持される。
- シールをめくった時、妖精がいたシールでは Unity Console にログが出る。
- シールをめくった時、妖精がいなかったシールでは妖精発見ログが出ない。
- 一度めくったシールで、同じ妖精発見ログが重複出力されない。
- 妖精判定は配置時のみ行われ、めくり時に新たな再抽選をしない。
