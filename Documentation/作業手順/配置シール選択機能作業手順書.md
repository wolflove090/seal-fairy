# 配置シール選択機能 作業手順書

## 目的
- シール配置フェーズ中に、画面左下へ所持シール一覧のスクロール UI を表示する。
- 一覧から選択したシールを、配置プレビューと実際の配置対象へ反映する。
- `StickerPeeling` フェーズでは一覧を非表示にし、`StickerPlacement` 再入時は未選択状態へ戻す。
- 所持シール一覧は Inspector の固定配列ではなく、所持品管理データを参照する構造で追加する。

## 完成イメージ
- 起動直後の `StickerPlacement` フェーズで、左下に所持シール一覧パネルが表示される。
- 一覧セルには画像が表示され、先頭シールが初期選択状態になっている。
- 一覧内の別シールをクリックまたはタップすると、選択ハイライトと配置プレビューが切り替わる。
- 選択中シールがある時だけ、ワールド上へそのシールを配置できる。
- `StickerPeeling` フェーズへ遷移すると一覧は非表示になる。
- `StickerPlacement` に戻ると選択が解除され、再度選ぶまで配置できない。

## 手順3: 選択状態管理クラスを追加する
1. `StickerSelectionState.cs` を新規作成する。
2. 役割は所持シール一覧、選択中シール、未選択状態の一元管理にする。
3. 起動直後の初期選択と、フェーズ再入時の選択解除を分けて扱える API を用意する。
4. 実装イメージは以下。

```csharp
using System.Collections.Generic;

public sealed class StickerSelectionState
{
    public IReadOnlyList<OwnedStickerDefinition> OwnedStickers { get; private set; }
    public OwnedStickerDefinition SelectedSticker { get; private set; }

    public void SetOwnedStickers(IReadOnlyList<OwnedStickerDefinition> ownedStickers)
    {
        OwnedStickers = ownedStickers;
    }

    public void SelectInitialSticker()
    {
        SelectedSticker = OwnedStickers != null && OwnedStickers.Count > 0 ? OwnedStickers[0] : null;
    }

    public void Select(OwnedStickerDefinition sticker)
    {
        SelectedSticker = sticker;
    }

    public void ClearSelection()
    {
        SelectedSticker = null;
    }
}
```

5. 同じセルを再度押した場合も未選択へ戻さないため、`Select` は常に上書きだけでよい。
6. 所持シール 0 件時は `SelectedSticker` を `null` にする。

## 手順6: HudScreenBinder に一覧生成と選択制御を追加する
1. [HudScreenBinder.cs](/Users/tatsuki/Projects/Unity/SealFairy/Assets/Scripts/HudScreenBinder.cs) を開く。
2. `UIDocument` から `sticker-scroll-view` と左下パネル root を取得できるようにする。
3. `OwnedStickerInventorySource` と `StickerSelectionState` を参照できるよう、SerializeField か初期化経路を追加する。
4. 起動時に所持シール一覧を読み込み、セルを動的生成する。
5. 初回だけ `SelectInitialSticker()` を呼び、UI ハイライトを更新する。
6. セル押下時の処理は次の順で行う。
   - `selectionState.Select(sticker)`
   - セルの選択見た目更新
   - `TapStickerPlacer` または共有状態へ選択変更を反映
7. `PhaseChanged` 受信時は以下の動作にする。
   - `StickerPlacement`: 左下一覧を表示する
   - `StickerPeeling`: 左下一覧を非表示にする
8. `StickerPlacement` へ戻ったタイミングでは `selectionState.ClearSelection()` を呼び、見た目も未選択状態へ戻す。
9. 起動直後の初期選択と、フェーズ再入時の選択解除を混同しないよう、初回初期化フラグを分ける。
10. 最低限の実装例は以下。

```csharp
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UIElements;

public sealed class HudScreenBinder : MonoBehaviour
{
    [SerializeField] private UIDocument uiDocument;
    [SerializeField] private OwnedStickerInventorySource inventorySource;

    private readonly StickerSelectionState selectionState = new();
    private readonly Dictionary<OwnedStickerDefinition, VisualElement> stickerCellByDefinition = new();

    private SealPhaseEventHub eventHub;
    private Button readyButton;
    private VisualElement stickerPanel;
    private ScrollView stickerScrollView;
    private Label emptyStickerListLabel;
    private bool isSubscribed;
    private bool hasAppliedInitialSelection;
    private SealGamePhase currentPhase = SealGamePhase.StickerPlacement;

    public StickerSelectionState SelectionState => selectionState;

    public void Initialize(SealPhaseEventHub eventHub)
    {
        UnsubscribeFromEventHub();
        this.eventHub = eventHub;
        SubscribeToEventHub();
        HandlePhaseChanged(SealGamePhase.StickerPlacement);
    }

    private void OnEnable()
    {
        VisualElement root = uiDocument.rootVisualElement;
        readyButton = root.Q<Button>("ready-button");
        stickerPanel = root.Q<VisualElement>("bottom-left-sticker-panel");
        stickerScrollView = root.Q<ScrollView>("sticker-scroll-view");
        emptyStickerListLabel = root.Q<Label>("empty-sticker-list-label");

        if (readyButton == null)
        {
            Debug.LogError("ready-button が見つかりません。");
            return;
        }

        if (stickerPanel == null || stickerScrollView == null)
        {
            Debug.LogError("シール一覧 UI が見つかりません。");
            return;
        }

        readyButton.clicked += HandleReadyButtonClicked;
        BuildStickerList();
        UpdateReadyButtonLabel();
        UpdateStickerPanelVisibility();
        SubscribeToEventHub();
    }

    private void OnDisable()
    {
        if (readyButton != null)
        {
            readyButton.clicked -= HandleReadyButtonClicked;
        }

        UnsubscribeFromEventHub();
    }

    private void BuildStickerList()
    {
        stickerCellByDefinition.Clear();
        stickerScrollView.Clear();

        IReadOnlyList<OwnedStickerDefinition> ownedStickers = inventorySource != null
            ? inventorySource.GetOwnedStickers()
            : null;

        selectionState.SetOwnedStickers(ownedStickers);

        if (ownedStickers == null || ownedStickers.Count == 0)
        {
            if (emptyStickerListLabel != null)
            {
                emptyStickerListLabel.style.display = DisplayStyle.Flex;
            }
            return;
        }

        if (emptyStickerListLabel != null)
        {
            emptyStickerListLabel.style.display = DisplayStyle.None;
        }

        foreach (OwnedStickerDefinition sticker in ownedStickers)
        {
            Button cell = CreateStickerCell(sticker);
            stickerCellByDefinition.Add(sticker, cell);
            stickerScrollView.Add(cell);
        }

        if (!hasAppliedInitialSelection)
        {
            selectionState.SelectInitialSticker();
            hasAppliedInitialSelection = true;
        }

        RefreshSelectionVisuals();
    }

    private Button CreateStickerCell(OwnedStickerDefinition sticker)
    {
        Button cell = new();
        cell.AddToClassList("sticker-cell");

        VisualElement image = new();
        image.AddToClassList("sticker-cell__image");
        if (sticker.Icon != null)
        {
            image.style.backgroundImage = new StyleBackground(sticker.Icon.texture);
        }

        cell.Add(image);
        cell.clicked += () => HandleStickerCellClicked(sticker);
        return cell;
    }

    private void HandleStickerCellClicked(OwnedStickerDefinition sticker)
    {
        selectionState.Select(sticker);
        RefreshSelectionVisuals();
    }

    private void RefreshSelectionVisuals()
    {
        foreach ((OwnedStickerDefinition sticker, VisualElement cell) in stickerCellByDefinition)
        {
            cell.EnableInClassList(
                "sticker-cell--selected",
                selectionState.SelectedSticker == sticker);
        }
    }

    private void HandleReadyButtonClicked()
    {
        eventHub?.RequestPhaseToggle();
    }

    private void HandlePhaseChanged(SealGamePhase phase)
    {
        bool returnedToPlacement =
            currentPhase == SealGamePhase.StickerPeeling &&
            phase == SealGamePhase.StickerPlacement;

        currentPhase = phase;
        UpdateReadyButtonLabel();
        UpdateStickerPanelVisibility();

        if (returnedToPlacement)
        {
            selectionState.ClearSelection();
            RefreshSelectionVisuals();
        }
    }

    private void UpdateStickerPanelVisibility()
    {
        if (stickerPanel == null)
        {
            return;
        }

        stickerPanel.style.display =
            currentPhase == SealGamePhase.StickerPlacement
                ? DisplayStyle.Flex
                : DisplayStyle.None;
    }

    private void UpdateReadyButtonLabel()
    {
        if (readyButton == null)
        {
            return;
        }

        readyButton.text =
            currentPhase == SealGamePhase.StickerPlacement
                ? "シールめくりへ"
                : "シール配置へ";
    }

    private void SubscribeToEventHub()
    {
        if (isSubscribed || eventHub == null || !isActiveAndEnabled)
        {
            return;
        }

        eventHub.PhaseChanged += HandlePhaseChanged;
        isSubscribed = true;
    }

    private void UnsubscribeFromEventHub()
    {
        if (!isSubscribed || eventHub == null)
        {
            return;
        }

        eventHub.PhaseChanged -= HandlePhaseChanged;
        isSubscribed = false;
    }
}
```

11. `SelectionState` を public getter で公開しておくと、`TapStickerPlacer` 側から同じ選択状態を参照しやすい。
12. `BuildStickerList()` は初回だけ先頭選択を適用し、フェーズ再入時には再選択しないよう `hasAppliedInitialSelection` で分ける。
13. `CreateStickerCell()` では `Button` を使うとクリック処理が簡単で、選択状態 class の切り替えも行いやすい。
14. `HandlePhaseChanged()` では `StickerPeeling -> StickerPlacement` の戻りだけを検出し、その時に未選択化する。
15. `empty-sticker-list-label` を UXML に置く場合は、所持 0 件時だけ `DisplayStyle.Flex` にする。

- `StickerPeeling` フェーズでは一覧が非表示になる。
- `StickerPlacement` 再入時は未選択状態に戻り、選択するまで配置されない。
- UI 操作中にワールドへ誤ってシールが配置されない。
- 所持シール一覧の取得口が所持品管理データ接続前提の構造になっている。

## 補足
- 所持品管理データの本実装がまだ未整備なら、仮データを `OwnedStickerInventorySource` に閉じ込めておき、`HudScreenBinder` と `TapStickerPlacer` からは直接参照しない。
- 画像表示は今回必須要件だが、画像のアスペクト比が揃っていない場合は CSS 側でトリミング方針を決めておくと UI 崩れを防ぎやすい。
- 初期選択と再入時未選択は意図が異なるため、`OnEnable` のような単一タイミングへまとめず、初回初期化とフェーズイベントを分けて扱う。
