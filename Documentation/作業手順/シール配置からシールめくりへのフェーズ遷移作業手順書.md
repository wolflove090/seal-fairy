# シール配置からシールめくりへのフェーズ遷移 作業手順書

## 目的
- 起動直後をシール配置フェーズにし、シール配置だけが実行できる状態を作る。
- 右上の既存 UI ボタン押下でシールめくりフェーズへ遷移できるようにする。
- シールめくりフェーズではシールめくりだけを許可し、シール配置は無効化する。
- シールめくりフェーズから配置フェーズへ戻る時、未めくりシールを全削除する。
- UI と進行管理はイベント経由で接続し、`HudScreenBinder` と `SealPhaseController` を疎結合に保つ。

## 完成イメージ
- Play 開始直後はシール配置ができ、既存シールをクリックしてもめくれない。
- 右上ボタンを押すと配置が無効になり、配置済みシールだけがめくれる。
- めくりフェーズ中に空いている場所をクリックしても新しいシールは生成されない。
- 再度右上ボタンを押すと配置フェーズに戻り、未めくりシールはすべて消える。
- 右上ボタン文言は現在フェーズに応じて切り替わる。

## 手順2: イベントハブを追加する
1. `Assets/Scripts/SealPhaseEventHub.cs` を新規作成する。
2. `MonoBehaviour` にはせず、plain C# class として作る。
3. 以下のコードを入れる。

```csharp
using System;

public sealed class SealPhaseEventHub
{
    public event Action PhaseToggleRequested;
    public event Action<SealGamePhase> PhaseChanged;

    public void RequestPhaseToggle()
    {
        PhaseToggleRequested?.Invoke();
    }

    public void NotifyPhaseChanged(SealGamePhase phase)
    {
        PhaseChanged?.Invoke(phase);
    }
}
```

## 手順4: SealPhaseController を追加する
1. `Assets/Scripts/SealPhaseController.cs` を新規作成する。
2. 役割は現在フェーズ管理、イベント購読、配置 / めくり入力切替、未めくりシール全削除。
3. 最低限、以下の責務を入れる。
   - `Initialize(SealPhaseEventHub eventHub)`
   - `OnEnable` / `OnDisable` でイベント購読と解除
   - 初期フェーズを `StickerPlacement` にする処理
   - フェーズ切替処理
   - `TapStickerPlacer` への配置可否反映
   - 全アクティブシールへのめくり可否反映
   - 戻る時の未めくりシール全削除
4. 実装イメージは以下。

```csharp
using UnityEngine;

public sealed class SealPhaseController : MonoBehaviour
{
    [SerializeField] private TapStickerPlacer tapStickerPlacer;

    public SealGamePhase CurrentPhase { get; private set; } = SealGamePhase.StickerPlacement;

    private SealPhaseEventHub eventHub;

    public void Initialize(SealPhaseEventHub eventHub)
    {
        this.eventHub = eventHub;
    }

    private void OnEnable()
    {
        if (eventHub != null)
        {
            eventHub.PhaseToggleRequested += HandlePhaseToggleRequested;
        }
    }

    private void Start()
    {
        ApplyPhase(SealGamePhase.StickerPlacement);
    }

    private void OnDisable()
    {
        if (eventHub != null)
        {
            eventHub.PhaseToggleRequested -= HandlePhaseToggleRequested;
        }
    }

    private void HandlePhaseToggleRequested()
    {
        if (CurrentPhase == SealGamePhase.StickerPlacement)
        {
            ApplyPhase(SealGamePhase.StickerPeeling);
            return;
        }

        ClearRemainingStickers();
        ApplyPhase(SealGamePhase.StickerPlacement);
    }

    private void ApplyPhase(SealGamePhase phase)
    {
        CurrentPhase = phase;
        tapStickerPlacer.SetPlacementEnabled(phase == SealGamePhase.StickerPlacement);

        foreach (PeelSticker3D sticker in StickerRuntimeRegistry.GetActiveStickers())
        {
            sticker.SetTapPeelEnabled(phase == SealGamePhase.StickerPeeling);
        }

        eventHub?.NotifyPhaseChanged(CurrentPhase);
    }

    private void ClearRemainingStickers()
    {
        foreach (PeelSticker3D sticker in StickerRuntimeRegistry.GetActiveStickers())
        {
            if (sticker != null)
            {
                Destroy(sticker.gameObject);
            }
        }

        StickerRuntimeRegistry.ClearAll();
    }
}
```

5. `Start` を使う場合、`Initialize` が `Awake` で先に呼ばれるように bootstrap の存在を前提にする。
6. `StickerRuntimeRegistry.GetActiveStickers()` は後の手順で追加する。

## 手順5: HudScreenBinder を追加する
1. `Assets/Scripts/HudScreenBinder.cs` を新規作成する。
2. このクラスは UI Toolkit にだけ依存させる。
3. `SealPhaseController` を直接参照しない。
4. 役割は次の 2 つだけに絞る。
   - `ready-button` 押下時に `eventHub.RequestPhaseToggle()` を呼ぶ
   - `PhaseChanged` を受けてボタン文言を更新する
5. 実装イメージは以下。

```csharp
using UnityEngine;
using UnityEngine.UIElements;

public sealed class HudScreenBinder : MonoBehaviour
{
    [SerializeField] private UIDocument uiDocument;

    private SealPhaseEventHub eventHub;
    private Button readyButton;

    public void Initialize(SealPhaseEventHub eventHub)
    {
        this.eventHub = eventHub;
    }

    private void OnEnable()
    {
        VisualElement root = uiDocument.rootVisualElement;
        readyButton = root.Q<Button>("ready-button");

        if (readyButton == null)
        {
            Debug.LogError("ready-button が見つかりません。");
            return;
        }

        readyButton.clicked += HandleReadyButtonClicked;

        if (eventHub != null)
        {
            eventHub.PhaseChanged += HandlePhaseChanged;
        }
    }

    private void OnDisable()
    {
        if (readyButton != null)
        {
            readyButton.clicked -= HandleReadyButtonClicked;
        }

        if (eventHub != null)
        {
            eventHub.PhaseChanged -= HandlePhaseChanged;
        }
    }

    private void HandleReadyButtonClicked()
    {
        eventHub?.RequestPhaseToggle();
    }

    private void HandlePhaseChanged(SealGamePhase phase)
    {
        readyButton.text = phase == SealGamePhase.StickerPlacement
            ? "シールめくりへ"
            : "シール配置へ";
    }
}
```

6. `uiDocument` は Inspector 参照で設定する。
7. `ready-button` の name は変えない。

## 手順8: StickerRuntimeRegistry を拡張する
1. [StickerRuntimeRegistry.cs](/Users/tatsuki/Projects/Unity/SealFairy/Assets/Scripts/StickerRuntimeRegistry.cs) を開く。
2. 妖精有無だけでなく、アクティブシール参照も持てるようにする。
3. 構造は `Dictionary<int, StickerRuntimeInfo>` でも、`Dictionary<int, PeelSticker3D>` と `Dictionary<int, bool>` の 2 本でもよい。最小差分なら 2 本でもよい。
4. 例として以下のように整理する。

```csharp
using System.Collections.Generic;

public static class StickerRuntimeRegistry
{
    private static readonly Dictionary<int, bool> fairyByStickerId = new();
    private static readonly Dictionary<int, PeelSticker3D> stickerById = new();

    public static void Register(PeelSticker3D sticker, bool hasFairy)
    {
        if (sticker == null)
        {
            return;
        }

        int key = sticker.GetInstanceID();
        fairyByStickerId[key] = hasFairy;
        stickerById[key] = sticker;
    }

    public static bool TryConsumeFairy(PeelSticker3D sticker, out bool hasFairy)
    {
        hasFairy = false;
        if (sticker == null)
        {
            return false;
        }

        int key = sticker.GetInstanceID();
        if (!fairyByStickerId.TryGetValue(key, out hasFairy))
        {
            return false;
        }

        fairyByStickerId.Remove(key);
        return true;
    }

    public static IReadOnlyCollection<PeelSticker3D> GetActiveStickers()
    {
        List<PeelSticker3D> results = new();
        foreach (PeelSticker3D sticker in stickerById.Values)
        {
            if (sticker != null)
            {
                results.Add(sticker);
            }
        }

        return results;
    }

    public static void Unregister(PeelSticker3D sticker)
    {
        if (sticker == null)
        {
            return;
        }

        int key = sticker.GetInstanceID();
        fairyByStickerId.Remove(key);
        stickerById.Remove(key);
    }

    public static void ClearAll()
    {
        fairyByStickerId.Clear();
        stickerById.Clear();
    }
}
```

5. `PeelSticker3D.OnDestroy()` の末尾か先頭で `StickerRuntimeRegistry.Unregister(this);` を呼ぶ。
6. これで自然破棄と強制削除の両方で参照を掃除できる。

## 動作確認時の着眼点
- `HudScreenBinder` が `SealPhaseController` を直接参照していないか。
- `SealPhaseEventHub` を複数箇所で `new` していないか。
- `OnEnable` の時点で `eventHub` が未注入だと購読できないので、初期化順が崩れていないか。
- 戻り時に `Destroy` と `OnDestroy` 経由の registry 掃除が二重でも破綻しないか。
- 新規生成シールがめくりフェーズでちゃんと `SetTapPeelEnabled(true)` を受けるか。

## 補足
- `TapStickerPlacer` がランタイム自動生成される構成なので、`SealPhaseController` からの参照解決は Inspector 固定参照よりランタイム取得の方が扱いやすい。

- `SealPhaseBootstrap` で生成した同一インスタンスを全員が共有することが重要。
