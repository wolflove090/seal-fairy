# シール妖精発見エフェクト 作業手順書

## 目的
- 配置したシールに 50% の確率で妖精を割り当てる。
- 妖精ありシールにのみ `KiraKiraEffect.prefab` をシール裏側へ仕込む。
- めくるまではエフェクトを見せず、めくり後に視認できる状態にする。
- シールはめくり完了後すぐには消さず、2 秒後に破棄する。
- ユーザーがそのまま実装できるよう、作業順と変更コードのイメージをまとめる。

## 完成イメージ
- シール配置時に `Random.value < 0.5f` で妖精有無が決まる。
- 妖精ありシールには、`KiraKiraEffect.prefab` が子オブジェクトとして追加される。
- エフェクトはシール裏側に隠れているため、未剥がし時には見えない。
- めくり完了時、妖精ありシールではキラキラが見え、ログも 1 回出る。
- シール本体は `Destroy(gameObject, 2f)` で 2 秒後に破棄され、子エフェクトも同時に消える。

## 手順1: エフェクト prefab を `Resources` から読める場所へ移す
1. Unity Editor 上で `KiraKiraEffect.prefab` を `Assets/Effect/Kirakira/Resources/` 配下へ移動する。
2. 移動後の想定パスを次にする。

```text
Assets/Effect/Kirakira/Resources/KiraKiraEffect.prefab
```

3. `.meta` を維持したまま移動できるよう、Unity Editor 上での移動を優先する。
4. この配置であれば、コード側は次のパスでロードできる。

```csharp
Resources.Load<GameObject>("KiraKiraEffect");
```

5. `Resources.Load<GameObject>("Effect/Kirakira/KiraKiraEffect")` ではない点に注意する。

## 手順2: 配置時にエフェクトを仕込む処理を追加する
1. [TapStickerPlacer.cs](/Users/tatsuki/Projects/Unity/SealFairy/Assets/Scripts/TapStickerPlacer.cs) を開く。
2. `SpawnSticker` の流れを「生成 -> 妖精判定 -> registry 登録 -> 妖精ありならエフェクト生成」に拡張する。
3. prefab を毎回 `Resources.Load` しないよう、private static cache を追加する。
4. 追加イメージは以下。

```csharp
using UnityEngine;

[DisallowMultipleComponent]
public sealed class TapStickerPlacer : MonoBehaviour
{
    private static GameObject cachedFairyEffectPrefab;

    private static GameObject LoadFairyEffectPrefab()
    {
        if (cachedFairyEffectPrefab != null)
        {
            return cachedFairyEffectPrefab;
        }

        cachedFairyEffectPrefab = Resources.Load<GameObject>("KiraKiraEffect");
        return cachedFairyEffectPrefab;
    }

    private static void AttachFairyEffect(PeelSticker3D sticker)
    {
        if (sticker == null)
        {
            return;
        }

        GameObject fairyEffectPrefab = LoadFairyEffectPrefab();
        if (fairyEffectPrefab == null)
        {
            Debug.LogWarning("KiraKiraEffect prefab を Resources から読み込めませんでした。");
            return;
        }

        GameObject fairyEffect = Instantiate(fairyEffectPrefab, sticker.transform);
        fairyEffect.name = fairyEffectPrefab.name;
        fairyEffect.transform.localPosition = Vector3.zero;
        fairyEffect.transform.localRotation = Quaternion.identity;
        fairyEffect.transform.localScale = Vector3.one;
    }
}
```

5. 次に `SpawnSticker` の末尾を以下の形にする。

```csharp
private void SpawnSticker(Vector3 worldPoint)
{
    PeelSticker3D sticker = Instantiate(templateSticker);
    sticker.name = "Peel Sticker";
    sticker.transform.SetPositionAndRotation(worldPoint, templateSticker.transform.rotation);
    sticker.transform.localScale = templateSticker.transform.localScale;
    sticker.gameObject.SetActive(true);
    sticker.PeelAmount = 0f;

    bool hasFairy = Random.value < 0.5f;
    StickerRuntimeRegistry.Register(sticker, hasFairy);

    if (hasFairy)
    {
        AttachFairyEffect(sticker);
    }
}
```

6. テンプレート用の非表示シールにはこの helper を呼ばないことを確認する。
7. 失敗時はエフェクト生成だけをスキップし、シール配置は継続するようにしておく。

## 手順3: めくり完了処理を 2 秒遅延破棄へ変更する
1. [PeelSticker3D.cs](/Users/tatsuki/Projects/Unity/SealFairy/Assets/Scripts/PeelSticker3D.cs) を開く。
2. 完了処理の重複実行を防ぐため、private field を 1 つ追加する。

```csharp
private bool isPeelComplete;
```

3. `Update` の入力受付と自動めくり進行を、完了後は通さないように調整する。
4. 変更イメージは以下。

```csharp
private void Update()
{
    if (!Application.isPlaying)
    {
        return;
    }

    if (allowTapPeel && !isPeelComplete)
    {
        HandlePointer();
    }

    if (isAutoPeeling && !isPeelComplete)
    {
        float nextAmount = Mathf.MoveTowards(peelAmount, 1f, Time.deltaTime / autoPeelDuration);
        PeelAmount = nextAmount;
        if (nextAmount >= 1f)
        {
            CompletePeel();
        }
    }
}
```

5. 続いて、完了処理を private method に分離する。

```csharp
private void CompletePeel()
{
    if (isPeelComplete)
    {
        return;
    }

    isPeelComplete = true;
    isAutoPeeling = false;

    if (StickerRuntimeRegistry.TryConsumeFairy(this, out bool hasFairy) && hasFairy)
    {
        Debug.Log("妖精を発見！");
    }

    Destroy(gameObject, 2f);
}
```

6. 既存の `Destroy(gameObject)` は削除する。
7. `Destroy(gameObject, 2f)` は 1 回だけ呼ぶようにし、毎フレーム呼ばない。

## 手順4: エフェクトの見え方を Unity 上で確認する
1. Play モードに入る。
2. シールを複数枚配置する。
3. Hierarchy で、妖精ありシールにのみ `KiraKiraEffect` が子として付与されることを確認する。
4. シール未剥がし状態で、キラキラが表側から見えていないことを確認する。
5. 見えてしまう場合は、まずシール裏面メッシュとの前後関係を確認する。
6. それでも必要なら、実装前提は `localPosition = Vector3.zero` のままにしつつ、後続の調整タスクとして切り出す。

## 手順5: ログと破棄タイミングを確認する
1. 妖精ありシールをめくる。
2. Console に `Debug.Log("妖精を発見！")` が 1 回だけ出ることを確認する。
3. キラキラが見えたまま、シールがすぐには消えないことを確認する。
4. 約 2 秒後にシールごと消えることを確認する。
5. 妖精なしシールでは、ログもキラキラも出ないことを確認する。
6. 初期デモシールをめくっても例外が出ないことを確認する。

## 作業時の着眼点
- `Resources.Load` の戻り値が `null` の場合、実ファイルの配置場所が間違っていないか。
- `KiraKiraEffect` が未剥がし時に見える場合、シール裏面とエフェクトの描画関係に問題がないか。
- `Destroy(gameObject, 2f)` を完了時に 1 回しか呼んでいないか。
- `TryConsumeFairy` を 2 回以上呼んでいないか。
- テンプレートシールや初期デモシールを誤ってエフェクト対象にしていないか。

## 完了条件
- `KiraKiraEffect.prefab` が `Resources.Load<GameObject>("KiraKiraEffect")` で取得できる場所にある。
- 配置した妖精ありシールにのみ `KiraKiraEffect` が子オブジェクトとして追加される。
- 未剥がし状態でエフェクトは見えず、めくり後に見える。
- 妖精ありシールでのみログが 1 回出る。
- シールはめくり完了後すぐに消えず、約 2 秒後に破棄される。
- 妖精なしシールと初期デモシールで不要演出や例外が発生しない。

## 補足
- 今回の手順では `StickerRuntimeRegistry` の API 追加は不要な前提で進めてよい。
- `Resources` 配下への移動は、参照切れ防止のため Unity Editor 上で実施する方が安全。
- `localPosition = Vector3.zero` で見え方に問題が出た場合でも、まずは設計どおり最小差分で実装し、その後に位置微調整を別タスクとして切る方がよい。
