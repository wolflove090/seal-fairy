# シール妖精配置と剥がしログ更新 作業手順書

## 目的
- シールを配置した時に 50% の確率で妖精を割り当てる。
- 妖精有無は `MonoBehaviour` 外の管理データで保持する。
- シールをめくり終わった時に、妖精がいた場合のみ Unity Console へログを出力する。
- ユーザーが手元でそのまま実装できるよう、変更箇所とコード例をまとめる。

## 完成イメージ
- クリックまたはタップで配置された各シールに、内部的に `true / false` の妖精フラグが付く。
- 妖精フラグはシール本体コンポーネントではなく、別の registry が持つ。
- シールをめくり切った時、妖精ありのシールだけ `Debug.Log` が 1 回出る。
- 妖精なしのシールではログが出ない。

## 手順1: 妖精管理用 registry を追加する
1. `Assets/Scripts` 配下に `StickerRuntimeRegistry.cs` を新規作成する。
2. まずは以下のコードを入れる。

```csharp
using System.Collections.Generic;

public static class StickerRuntimeRegistry
{
    private static readonly Dictionary<int, bool> fairyByStickerId = new();

    public static void Register(PeelSticker3D sticker, bool hasFairy)
    {
        if (sticker == null)
        {
            return;
        }

        fairyByStickerId[sticker.GetInstanceID()] = hasFairy;
    }

    public static bool TryConsumeFairy(PeelSticker3D sticker, out bool hasFairy)
    {
        hasFairy = false;
        if (sticker == null)
        {
            return false;
        }

        int key = sticker.GetInstanceID();
        if (!fairyByStickerId.TryGetValue(key, out hasFairy))
        {
            return false;
        }

        fairyByStickerId.Remove(key);
        return true;
    }
}
```

3. この registry は `PeelSticker3D` 自体に妖精情報を持たせず、インスタンス ID と `bool` を対応付けるために使う。
4. `TryConsumeFairy` は「参照したら削除する」実装なので、同じシールでログが重複しにくい。

## 手順2: 配置時に 50% 判定して registry へ登録する
1. [TapStickerPlacer.cs](/Users/tatsuki/Projects/Unity/SealFairy/Assets/Scripts/TapStickerPlacer.cs) を開く。
2. `SpawnSticker` を、生成したシールを登録できる形へ修正する。
3. 最小差分なら、`SpawnSticker` の末尾に登録処理を足せばよい。
4. 変更後のイメージは以下。

```csharp
private void SpawnSticker(Vector3 worldPoint)
{
    PeelSticker3D sticker = Instantiate(templateSticker);
    sticker.name = "Peel Sticker";
    sticker.transform.SetPositionAndRotation(worldPoint, templateSticker.transform.rotation);
    sticker.transform.localScale = templateSticker.transform.localScale;
    sticker.gameObject.SetActive(true);
    sticker.PeelAmount = 0f;

    bool hasFairy = Random.value < 0.5f;
    StickerRuntimeRegistry.Register(sticker, hasFairy);
}
```

5. `Random.value < 0.5f` が 50% 判定になる。
6. 判定は配置時に 1 回だけ行う。めくり時に再抽選しない。
7. 登録対象は `Instantiate` した実体のみで、非表示テンプレートは登録しない。

## 手順3: めくり完了時に registry を参照してログを出す
1. [PeelSticker3D.cs](/Users/tatsuki/Projects/Unity/SealFairy/Assets/Scripts/PeelSticker3D.cs) を開く。
2. `Update` の `isAutoPeeling` 分岐の中にある、完了時の `Destroy(gameObject)` 直前を修正する。
3. 変更イメージは以下。

```csharp
if (isAutoPeeling)
{
    float nextAmount = Mathf.MoveTowards(peelAmount, 1f, Time.deltaTime / autoPeelDuration);
    PeelAmount = nextAmount;
    if (nextAmount >= 1f)
    {
        if (StickerRuntimeRegistry.TryConsumeFairy(this, out bool hasFairy) && hasFairy)
        {
            Debug.Log("妖精がいました");
        }

        Destroy(gameObject);
    }
}
```

4. ログ出力タイミングは、めくり開始時ではなく「めくり完了時」にする。
5. `TryConsumeFairy` が失敗した場合は未登録シールなので、そのまま `Destroy(gameObject)` してよい。
6. ログ文言は `"妖精がいました"` のように、妖精がいた事実が分かればよい。

## 手順4: 必要なら後始末を追加する
1. 今回の最小構成では、通常は `TryConsumeFairy` で辞書から消えるため十分。
2. ただし、別経路でシールが破棄される可能性が気になる場合は `Unregister` を追加してもよい。
3. 追加するなら registry に以下を足す。

```csharp
public static void Unregister(PeelSticker3D sticker)
{
    if (sticker == null)
    {
        return;
    }

    fairyByStickerId.Remove(sticker.GetInstanceID());
}
```

4. この対応は保険なので、現状の動作確認で問題がなければ必須ではない。

## 手順5: 動作確認
1. Play モードに入る。
2. シールを複数枚配置する。
3. それぞれのシールを順にクリックしてめくる。
4. Console を開き、妖精ありのシールでのみログが出ることを確認する。
5. 妖精なしのシールではログが出ないことを確認する。
6. 同じシールで同一ログが重複しないことを確認する。
7. めくり後にシール自体は既存どおり消えることを確認する。

## 動作確認時の着眼点
- 配置したシールはすべて通常どおり生成されるか。
- テンプレート用シールが誤ってログ対象になっていないか。
- 初期配置のデモシールをめくった時、未登録でもエラーにならないか。
- Console に不要なログが増えていないか。

## 完了条件
- 配置時に毎回 50% 判定が走る。
- 妖精有無は `PeelSticker3D` のフィールドではなく外部 registry に保持される。
- めくり完了時にのみ妖精有無が参照される。
- 妖精ありのシールだけログが出る。
- 妖精なしのシールではログが出ない。
- 同じシールでログが 2 回以上出ない。

## 補足
- 今回の構成は最小差分優先なので、後でテストしやすくしたくなったら `Random` やログ出力を抽象化してよい。
- 現時点では「ユーザーが配置したシール」が主対象なので、デモ起動時の初期シールをログ対象外として扱っても問題ない。
