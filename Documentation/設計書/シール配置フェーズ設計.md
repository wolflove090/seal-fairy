# シール配置フェーズ設計

## 目的
- `SealPlacement` フェーズで、プレイヤーがカメラを移動しながらシール配置位置を決め、残数が尽きるまでシールを配置できるようにする。
- Unity 固有処理を入力取得、座標変換、描画反映に限定し、配置ルールとフェーズ進行はドメイン層で管理する。

## 設計方針
- フェーズ中のゲーム状態は `PlacementPhaseDomainService` が保持し、`MonoBehaviour` は毎フレームの入力スナップショットを渡すだけにする。
- 画面座標からワールド座標への変換は Unity アダプタ層で行い、ドメイン層へは Unity 型を避けた `WorldPoint` を渡す。
- 配置候補プレビュー表示と残数表示は UI / View 層で行い、表示可否や表示値はドメインの状態から決定する。

## ユースケース
1. フェーズ開始時に残り配置数 10 の状態で配置コンテキストを初期化する。
2. プレイヤーがポインタを動かすと、プレビューシールが現在のワールド座標へ追従する。
3. プレイヤーがポインタ押下した時点で残数があればシールを 1 枚生成する。
4. シール生成時に妖精を内包するかを 50% の確率で決定し、シール状態へ保存する。
5. シール配置後に残数を減算し、プレビュー表示へ即時反映する。
6. 右下ボタン押下で剥がしフェーズへ遷移する。

## レイヤ分割

### ドメイン層
- `GamePhase`
  - フェーズ種別を表す enum。
- `StickerId`
  - シール個体識別子。
- `WorldPoint`
  - Unity 非依存の座標値。
- `PlacedSticker`
  - `StickerId`、配置座標、妖精有無、現在状態を保持するエンティティ。
- `StickerCollection`
  - 配置済みシール一覧を管理するコレクション。
- `PlacementPhaseState`
  - 残数、プレビュー座標、配置可能状態を保持する値オブジェクト。
- `IStickerRandomService`
  - 妖精内包判定を返すインターフェース。
- `PlacementPhaseDomainService`
  - 配置入力を受けて、残数減算、シール追加、プレビュー更新を行う。

### アプリケーション層
- `GameFlowService`
  - 現在フェーズと全体状態を管理し、配置フェーズと剥がしフェーズのサービスを切り替える。
- `PlacementPhaseUseCase`
  - 入力スナップショットを解釈して `PlacementPhaseDomainService` を呼び出し、ViewModel を組み立てる。
- `PlacementViewModel`
  - プレビュー表示有無、プレビュー座標、残数表示文言、完了ボタン文言を UI 層へ渡す。

### Unity アダプタ層
- `PointerInputAdapter`
  - マウスとタッチを統一して入力スナップショット化する。
- `WorldPlaneRaycaster`
  - 画面座標を操作平面上の `WorldPoint` に変換する。
- `PlacementPreviewPresenter`
  - プレビューオブジェクトの位置更新と表示切替を行う。
- `StickerSpawner`
  - ドメインが生成要求した `PlacedSticker` を Unity 上のシール表示オブジェクトへ変換する。
- `GameHudPresenter`
  - UI Toolkit のラベルとボタン文言を更新する。

## 主要フロー
1. `PointerInputAdapter` がマウスまたはタッチから共通入力を生成する。
2. `WorldPlaneRaycaster` がポインタ位置を `WorldPoint` に変換する。
3. `PlacementPhaseUseCase` が現在フェーズを確認し、配置フェーズであればプレビュー座標更新を要求する。
4. クリック確定時に `PlacementPhaseDomainService.TryPlaceSticker` を実行する。
5. ドメインは残数確認後、`PlacedSticker` を生成し、`StickerCollection` に追加する。
6. `IStickerRandomService` が妖精有無を決定する。
7. アプリケーション層は生成イベントを `StickerSpawner` へ渡し、Unity オブジェクトを生成する。
8. `GameHudPresenter` と `PlacementPreviewPresenter` が最新 ViewModel を反映する。

## 状態遷移
- 開始時: `remainingCount = 10`
- 配置成功時: `remainingCount -= 1`
- `remainingCount == 0` でもフェーズは継続し、プレビューは表示したまま配置要求のみ拒否する。
- 完了ボタン押下時: `GameFlowService` が `SealPeeling` へ遷移する。

## 入出力設計

### 入力
- `PointerSnapshot`
  - `screenPosition`
  - `pressedThisFrame`
  - `releasedThisFrame`
  - `isPointerOverUi`
- `WorldPoint? currentPointerWorld`
- `bool completePhaseRequested`

### 出力
- `PlacementPhaseResult`
  - `PlacedSticker? createdSticker`
  - `PlacementViewModel viewModel`
  - `bool phaseCompleted`

## 依存ルール
- ドメイン層は `UnityEngine` 名前空間を参照しない。
- アプリケーション層はドメイン層のみ参照し、Unity API を直接呼ばない。
- Unity アダプタ層のみ `MonoBehaviour` と `UnityEngine` を利用する。
- ドメインイベントの通知は C# の戻り値またはイベントデータで行い、 `GameObject` を直接扱わない。

## リスクと対策
- 入力座標が UI と競合する可能性がある。
  - `PointerInputAdapter` で UI ヒット時は配置入力を無効化する。
- 残数 0 時の UX が不明瞭になりうる。
  - ボタン文言と残数ラベルを更新し、配置不可状態を明示する。
- 配置済みデータと Unity 表示の整合性が崩れる可能性がある。
  - `StickerId` をキーにドメインと表示オブジェクトを一対一対応させる。
