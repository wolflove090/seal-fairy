# 配置シールマウス追従プレビュー設計

## 目的
- `SealPlacement` フェーズ中、選択中シールの配置予定位置をプレイヤーへ事前に示すため、`TapStickerPlacer` にマウス追従プレビューを追加する。
- プレビュー近傍へ選択中シールの残数を同時表示し、配置対象と在庫状況を一目で確認できるようにする。
- 既存のシール選択 UI、在庫減算、フェーズ切り替え処理と整合しつつ、責務が過度に集中しない構成にする。

## 設計方針
- ワールド座標に追従するシール見た目は `TapStickerPlacer` が管理し、選択中シールに応じたプレビュー prefab の差し替えと位置更新を担当する。
- 残数表示は UI Toolkit ではなくワールド追従のスクリーン座標表示とし、`TapStickerPlacer` が求めたプレビュー位置を HUD 側へ通知して表示する。
- 在庫値の真実のソースは引き続き `OwnedStickerInventorySource` とし、表示更新は `OwnedStickersChanged` と選択変更イベントの両方を契機に同期する。
- `HubScreenBinder` は既存の一覧 UI に加えて、カーソル近傍の残数表示ラベルの生成・更新・表示切り替えを担当する。

## 変更対象
- `Assets/Scripts/Sticker/TapStickerPlacer.cs`
- `Assets/Scripts/HudScreenBinder.cs`

## コンポーネント責務

### `StickerSelectionState`
- 選択中シールの保持に加えて、選択変更通知を発火する。
- `Select` と `ClearSelection` のタイミングでイベントを発行し、`TapStickerPlacer` と `HubScreenBinder` が受け取れるようにする。
- 同一シール再選択時は状態を維持しつつ、不要な再構築を避けるためイベント発火要否を制御する。

### `TapStickerPlacer`
- 既存の配置入力処理に加え、選択中シール prefab を元にしたプレビューインスタンスを 1 つだけ保持する。
- 毎フレーム、現在のマウス位置から配置平面上のワールド座標を求め、プレビューインスタンスをその座標へ移動する。
- 表示可否は「配置フェーズ中」「選択中シールあり」「在庫数 1 以上」「配置面ヒット成功」を条件に決める。
- プレビュー位置のスクリーン座標を算出し、残数表示 presenter へ渡すための通知または参照 API を提供する。
- 配置成功後は既存どおり `inventorySource.RemoveOwnedSticker(selectedSticker)` を実行し、その結果に応じてプレビュー表示を更新する。

### `HubScreenBinder`
- 左下の一覧 UI は現行責務を維持する。
- 追加で、画面上に浮かぶ残数表示ラベルを管理する。
- `StickerSelectionState` と `OwnedStickerInventorySource` の変化を受け、選択中シールの残数文言を更新する。
- `TapStickerPlacer` から受け取るスクリーン座標を基に、残数表示ラベルをカーソル近傍へ再配置する。
- フェーズ遷移時と未選択時は残数表示を非表示にする。

## データフロー
1. `HubScreenBinder` で一覧セルを押下すると `StickerSelectionState.Select(sticker)` が呼ばれる。
2. `StickerSelectionState` が選択変更イベントを通知する。
3. `TapStickerPlacer` が選択中シール prefab の変更を検知し、プレビュー用 `PeelSticker3D` を必要時のみ再生成する。
4. `TapStickerPlacer.Update` が毎フレーム `Input.mousePosition` を取得し、`TryGetPlaneHitPoint` でワールド座標へ変換する。
5. `TapStickerPlacer` がプレビュー位置を更新し、対応するスクリーン座標と表示可否を HUD 側へ通知する。
6. `HubScreenBinder` が選択中シールの残数を `inventorySource.GetOwnedStickerCount(selectedSticker)` から取得し、残数ラベル文言と位置を更新する。
7. 配置成功時は `TapStickerPlacer` がシールを生成し、直後に `OwnedStickerInventorySource.RemoveOwnedSticker` を呼ぶ。
8. `OwnedStickerInventorySource.OwnedStickersChanged` を受けた `HubScreenBinder` が一覧と残数表示を再計算する。
9. 残数が 0 になった場合、`BuildStickerList` により選択が解除され、`TapStickerPlacer` と `HubScreenBinder` がともにプレビュー／残数表示を消す。

## 表示設計

### プレビューシール
- `TapStickerPlacer` が保持する `templateSticker` を「非表示テンプレート」ではなく「プレビュー実体」として再利用する。
- 選択中 prefab の変更時のみ `Instantiate` / `Destroy` を行い、毎フレームの生成破棄はしない。
- プレビューは通常の配置済みシールと区別するため、`SetTapPeelEnabled(false)`、`PeelAmount = 0f`、必要なら半透明化または専用ソート設定を適用する。
- マウス位置が配置面に乗らない場合はプレビューを非表示にする。

### 残数表示ラベル
- `HudScreen.uxml` にプレビュー用オーバーレイ要素を追加する。
- ルート直下に `preview-count-label` を追加し、`position: absolute;` で画面座標へ直接配置する。
- ラベル位置はマウスカーソルやプレビューに重ならないよう、X/Y に固定オフセットを加える。
- 表記は `残り xN` もしくは同等の短い文言とする。

## イベント設計

### `StickerSelectionState`
- `event Action<StickerDefinition> SelectedStickerChanged` を追加する。
- `Select` で選択対象が変わった時のみ通知する。
- `ClearSelection` では `null` 通知を行い、購読側が非表示へ切り替えられるようにする。

### `TapStickerPlacer`
- `event Action<Vector2, bool> PreviewScreenPointChanged` のような通知を追加する。
- 引数は残数ラベルの配置先スクリーン座標と表示可否を表す。
- プレビュー非表示条件では `bool = false` を通知し、HUD が座標を保持したまま表示だけ切る挙動を選べるようにする。

## 主要処理

### プレビュー更新
1. `Update` 冒頭で `Application.isPlaying`、`isPlacementEnabled`、`selectionState.SelectedSticker` を確認する。
2. 選択中シールの prefab に対応するプレビューを `CacheTemplateSticker` で準備する。
3. `Input.mousePosition` を取得し、`TryGetPlaneHitPoint` でワールド座標へ変換する。
4. 在庫数が 1 以上でヒット成功した場合のみプレビューオブジェクトをアクティブ化し、座標更新する。
5. メインカメラで `WorldToScreenPoint` を行い、残数ラベル配置座標を算出して通知する。

### 配置確定
1. 既存の `TryGetPointerDownPosition` が成立したら、UI 上かどうか、シール上かどうか、配置面ヒット可否を判定する。
2. 配置成功後、選択中シールの在庫を減算する。
3. 在庫減算通知を受けて一覧 UI と残数表示が更新される。
4. 同一シールの残数が残っていればプレビューは継続、残数 0 または選択解除なら非表示にする。

### フェーズ切り替え
- `SealPhaseController` から `TapStickerPlacer.SetPlacementEnabled(false)` が呼ばれた際、プレビューを即時非表示にし、残数ラベル非表示通知も送る。
- `HubScreenBinder.HandlePhaseChanged` では既存の一覧表示切り替えに加え、プレビュー残数ラベルの表示可否も同期する。

## UI レイアウト詳細
- `preview-count-label` は `root` 直下に置き、ほかの固定 HUD より前面へ表示する。
- スタイルは暗色半透明背景、白文字、角丸の小型チップとする。
- 左下一覧と干渉しないよう、カーソルが画面端に近い場合はラベル位置を画面内に clamp する。
- モバイル向け分岐は追加しない。

## リスクと対策
- `TapStickerPlacer` に UI 表示責務まで載せると肥大化する。
  - ワールド座標計算とプレビュー表示に限定し、残数文言の決定とラベル描画は `HubScreenBinder` に残す。
- 選択解除と在庫変動の順序次第で一瞬だけ古い残数が出る可能性がある。
  - `OwnedStickersChanged` 発火後に `BuildStickerList` と残数表示更新を同フレームで実施し、選択状態を再評価する。
- マウスが画面外、または配置面ヒット不可のときに残数ラベルだけ残る可能性がある。
  - `TapStickerPlacer` 側のプレビュー非表示通知を単一の表示可否ソースにする。

## 検証方針
- 手動確認 1: `SealPlacement` 中にシール選択後、マウス移動でプレビューが追従する。
- 手動確認 2: プレビュー近傍の残数表示が選択中シールの在庫数と一致する。
- 手動確認 3: 同じシールを連続配置すると残数表示が即時減少する。
- 手動確認 4: 残数が 0 になるとプレビューと残数表示が消える。
- 手動確認 5: 別シール選択時にプレビュー見た目と残数が同時に切り替わる。
- 手動確認 6: `SealPeeling` 遷移でプレビューと残数表示が消え、`SealPlacement` 再入時は未選択なら表示されない。

## コードスニペット
```csharp
public sealed class StickerSelectionState
{
    public event Action<StickerDefinition> SelectedStickerChanged;

    public StickerDefinition SelectedSticker { get; private set; }

    public void Select(StickerDefinition sticker)
    {
        if (SelectedSticker == sticker)
        {
            return;
        }

        SelectedSticker = sticker;
        SelectedStickerChanged?.Invoke(SelectedSticker);
    }

    public void ClearSelection()
    {
        if (SelectedSticker == null)
        {
            return;
        }

        SelectedSticker = null;
        SelectedStickerChanged?.Invoke(null);
    }
}
```

```csharp
private void UpdatePreview()
{
    StickerDefinition selectedSticker = selectionState?.SelectedSticker;
    int ownedCount = inventorySource != null ? inventorySource.GetOwnedStickerCount(selectedSticker) : 0;

    if (!isPlacementEnabled || selectedSticker == null || ownedCount <= 0)
    {
        SetPreviewVisible(false);
        return;
    }

    if (!TryGetPointerPosition(out Vector3 screenPoint) ||
        !templateSticker.TryGetPlaneHitPoint(GetActiveCamera(), screenPoint, out Vector3 worldPoint))
    {
        SetPreviewVisible(false);
        return;
    }

    templateSticker.gameObject.SetActive(true);
    templateSticker.transform.position = worldPoint;
    NotifyPreviewScreenPoint(screenPoint, true);
}
```
