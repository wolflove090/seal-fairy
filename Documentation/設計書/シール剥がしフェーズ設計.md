# シール剥がしフェーズ設計

## 目的
- `SealPeeling` フェーズで、配置済みシールの剥がし入力、妖精判定ログ、剥がし完了後の削除、およびフェーズ終了時の全消去を成立させる。
- 既存 `PeelSticker3D` の見た目やメッシュ生成知見は参考にしつつ、剥がし進行とゲーム状態は Unity 非依存のドメインモデルとして扱う。

## 設計方針
- 剥がし対象の判定と状態更新は `PeelingPhaseDomainService` が担当する。
- 剥がし演出は `StickerPeelPresenter` が担当し、ドメインは「剥がし開始」「剥がし完了」「妖精発見」の事実だけを返す。
- フェーズ終了時の未剥がしシール全消去は `GameFlowService` の遷移処理で一括実施する。

## ユースケース
1. 剥がしフェーズ開始時に、配置済みシール一覧を剥がし対象として固定する。
2. プレイヤーがシールを選択すると、そのシールが未処理なら剥がし開始状態へ変わる。
3. View 層が剥がし演出を再生する。
4. 演出完了通知を受けると、対象シールを削除済みとして確定する。
5. 対象シールに妖精がいた場合のみログ文言を生成する。
6. 右下ボタン押下で、残っている未剥がしシールを全削除して配置フェーズへ戻る。

## レイヤ分割

### ドメイン層
- `StickerStatus`
  - `Placed`, `Peeling`, `Removed` を表す enum。
- `PeelingRequest`
  - どの `StickerId` を剥がすかを表す要求。
- `PeelingPhaseDomainService`
  - 剥がし開始可否判定、重複操作防止、完了時の削除確定を行う。
- `PeelingPhaseResult`
  - 剥がし開始、剥がし完了、妖精発見ログの有無、フェーズ完了可否を返す。
- `StickerLogMessageFactory`
  - ドメインイベントからログ文言を組み立てる。

### アプリケーション層
- `PeelingPhaseUseCase`
  - ポインタで選択された `StickerId` を解釈してドメインを実行する。
- `PeelingViewModel`
  - 完了ボタン文言、現在フェーズ文言、必要ならログ通知を UI へ渡す。
- `StickerRegistry`
  - `StickerId` と Unity 側表示オブジェクトを紐付ける。

### Unity アダプタ層
- `StickerHitTestAdapter`
  - レイキャストから選択されたシールの `StickerId` を取得する。
- `StickerPeelPresenter`
  - 剥がしアニメーションを実行し、完了時にアプリケーション層へ通知する。
- `UnityDebugLogger`
  - ログ文言を `Debug.Log` へ流す。
- `StickerCleanupPresenter`
  - 剥がし完了済み、またはフェーズ終了時一括削除のシール表示を破棄する。

## 主要フロー
1. `StickerHitTestAdapter` がクリック対象の `StickerId` を返す。
2. `PeelingPhaseUseCase` が `PeelingPhaseDomainService.TryStartPeeling` を呼び出す。
3. ドメインが `Placed` 状態なら `Peeling` に更新し、剥がし開始イベントを返す。
4. `StickerPeelPresenter` が対象の剥がし演出を再生する。
5. 演出完了時に `PeelingPhaseUseCase.CompletePeel` を実行する。
6. ドメインが対象を `Removed` にし、妖精ありならログイベントを返す。
7. `UnityDebugLogger` がログ出力し、`StickerCleanupPresenter` が表示オブジェクトを削除する。
8. フェーズ完了ボタン押下時に `GameFlowService` が未削除シールを全破棄して `SealPlacement` を再初期化する。

## 状態遷移
- `Placed -> Peeling`
  - クリックで剥がし開始。
- `Peeling -> Removed`
  - 演出完了通知で削除確定。
- `Placed -> Removed`
  - フェーズ終了時の一括消去。
- `Removed`
  - 再操作不可。

## 入出力設計

### 入力
- `StickerId? selectedStickerId`
- `bool completePhaseRequested`
- `StickerId? peelCompletedStickerId`

### 出力
- `PeelingPhaseResult`
  - `StickerId? startedPeelingStickerId`
  - `StickerId? removedStickerId`
  - `string? foundFairyLog`
  - `bool phaseCompleted`
  - `IReadOnlyList<StickerId> forceRemovedStickerIds`

## 既存 `PeelSticker3D` の扱い
- 既存スクリプトは以下の知見の参照元として使う。
  - シールの前面、背面、影メッシュ生成
  - 剥がし量に応じたカール形状更新
  - 破棄時の生成リソース解放
- 既存スクリプトの直接流用は行わず、演出専用コンポーネントとして責務を再定義する。
- ドメインは剥がし量や `Mesh` を知らず、演出進行は Presenter 内で閉じる。

## 依存ルール
- ドメイン層はシール選択結果として `StickerId` のみを扱い、 `Collider` や `RaycastHit` を持ち込まない。
- ログ文言生成はドメインまたはアプリケーション層で行い、 `Debug.Log` 呼び出しはアダプタ層へ限定する。
- 一括削除は `GameFlowService` のフェーズ遷移処理で行い、各 Presenter の個別判断に依存しない。

## リスクと対策
- 剥がし演出中の再クリックで状態不整合が起きる可能性がある。
  - `Peeling` 状態では再度 `TryStartPeeling` を拒否する。
- ドメイン削除と Unity オブジェクト削除のタイミングがずれる可能性がある。
  - 演出完了コールバックを唯一の削除確定トリガーにする。
- フェーズ終了時の全削除漏れが起きる可能性がある。
  - `StickerCollection` から残存 `StickerId` を列挙し、表示層へまとめて破棄指示を渡す。
